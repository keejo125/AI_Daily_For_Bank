## Hermes Agent

### 产品定位
Nous Research 2026年2月推出的开源Agent平台，被称为「会跟着你成长的Agent」。[1]

| 指标 | 数据 |
|------|------|
| GitHub Stars | 4万+ |
| 当前版本 | v0.8.0 |
| 更新频率 | 平均不到一周一个大版本 |
| 贡献者 | 240+ |
| 合并PR | 1400+ |

### 核心特性

| 特性 | 说明 |
|------|------|
| 自主Agent | 运行在你自己的服务器上 |
| 多平台接入 | Telegram、Discord、Slack、飞书、企业微信 |
| 内置学习闭环 | 记忆→技能→训练数据三层 |
| 定时自动化 | 自然语言cron指令 |
| Agent联邦 | 支持与OpenClaw互操作 |

### 三层学习闭环

| 层级 | 说明 | 价值 |
|------|------|------|
| 记忆 | MEMORY.md + USER.md + FTS5检索 | 跨会话上下文 |
| 技能 | 从经验中自动创建skill | 复用积累 |
| 训练数据 | 工具调用记录用于模型训练 | 越用越强 |

**核心公式**：
```
记忆沉淀 → 自动提炼技能 → 生成训练数据 → 模型能力提升 → 更强Agent
```

### 版本演进路线

| 版本 | 日期 | 主题 | 重点 |
|------|------|------|------|
| v0.5.0 | 2026-03-28 | hardening | 50+安全修复 |
| v0.7.0 | 2026-04-03 | resilience | 可插拔记忆、凭证轮换 |
| v0.8.0 | 最新 | intelligence | 后台任务通知、模型切换 |

**核心判断**：长期运行才是Agent真正的工程挑战（不是"不够聪明"，而是"跑着跑着崩了"）

## Qoder（通义灵码）

### 产品定位
阿里巴巴出品的AI编程工具（VS Code / JetBrains 插件 + 终端 CLI），定位"智能研发平台"，覆盖编码、测试、终端命令等场景。

### 终端沙箱能力

Qoder 实现了原生的"终端沙箱"能力，在 Agent 执行命令前做三层防护：

| 层级 | 机制 | 说明 |
|------|------|------|
| **LLM 风险判定** | 模型生成命令时判断意图 | 识别"删除系统路径"等高风险意图 |
| **AST 命令解析** | PowerShell/Bash/CMD 三套解析器 | 识别反引号注入、变量展开等隐蔽攻击 |
| **危险命令名单** | 内置名单比对 | 命中→弹窗确认；前两层命中→直接进沙箱 |

**跨平台沙箱实现**：
- **macOS**：sandbox-exec（Seatbelt），启动开销 +43-72ms
- **Linux**：bubblewrap，根目录只读，网络隔离
- **Windows**：自研 sandbox.exe（Rust），CreateRestrictedToken + DACL + Job Object，兼容 Win7+

**关键设计**：命令包裹（Command Wrapping）—— 原始命令被沙箱工具包裹后进入终端，Agent 看到被隔离的进程，用户看到透明。

### 核心能力矩阵

| 能力 | Qoder | Claude Code | 状态 |
|------|-------|-------------|------|
| IDE插件（VS Code/JetBrains） | ✅ | ❌ 客户端工具 | Qoder优势 |
| 终端CLI（qodercli） | ✅ | ✅ | 持平 |
| 终端沙箱（原生） | ✅ 三层防护 | ⚠️ Seatbelt/bwrap | Qoder优势 |
| 终端Agent执行 | ✅ 支持 | ✅ 支持 | 持平 |
| 多Agent协作 | ⚠️ | ⚠️ | 均弱 |
| 企业级策略 | ✅ 支持 | ⚠️ | Qoder优势 |

### 关键启示

### 对我行的价值

1. **Qoder终端沙箱可借鉴**：三层防护机制（LLM判定→AST解析→名单比对）是目前最完整的终端安全方案
2. **三平台覆盖是标配**：我行开发环境 macOS/Linux/Windows 并存，Qoder 的平台适配思路值得参考
3. **AST级解析 > 字符串黑名单**：能识别嵌套变形攻击，比传统安全方案更可靠
4. **Hermes开源路径**：长期运行稳定性比单次任务完成率更重要

### 风险点

1. **Qoder数据安全**：终端沙箱数据流向需评估（阿里云环境）
2. **Hermes运维成本**：7x24运行需要专业运维
3. **厂商锁定**： Nous Research / 阿里背景需评估
