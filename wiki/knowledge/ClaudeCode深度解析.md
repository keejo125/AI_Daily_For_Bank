## 核心设计理念

### 1. 后台记忆整合（KAIROS/autoDream）
当用户空闲时，后台启动守护进程执行"记忆整合"。

**类比**：像人类睡眠中大脑自动编排记忆——保留重要的，丢掉可以忘记的，整合重复的。

**运行方式**：
- 在名为 Undercover 的独立进程中运行
- 不依赖用户打开终端
- 作为操作系统守护进程运行

**价值**：用户第二天打开终端时，Claude Code 已经准备好整洁的起点，不需要重新阅读历史对话。[3]

### 2. 正反馈记忆
Claude Code 不仅记"负反馈"，也记"正反馈"——如果只记批评不记表扬，模型会趋向保守，不敢做决定。[3]

### 3. 成本控制：Prompt Cache Fork
多 Agent 协同工作时 token 消耗指数级增长。Claude Code 采用 fork 模式：
- 创建与父 Agent 字节相同的上下文副本
- 所有并行 Agent 命中同一份缓存
- 5个并行 Agent 的 token 成本 ≈ 1个 Agent 顺序工作

### 4. 安全优先（Fail-closed）
安全优先设计，后台进程隔离运行（Undercover 模式）。[3]

## 工程启示

### 对智能研发工具建设的启示

1. **记忆分层是基础**
   - 底层索引（始终加载）
   - 中间层主题（按需加载）
   - 上层原始对话（通过 grep 检索）

2. **后台整合提升体验**
   - KAIROS 模式——空闲时自动整理知识
   - 用户每次打开都有"整洁的起点"

3. **多 Agent 协作参考架构**
   - SubAgent：适合隔离复杂任务，降低上下文溢出风险
   - AgentTeams：适合多角色协作，Team Lead + Teammates 模式

4. **成本控制不可忽视**
   - Prompt Cache Fork 是降低多 Agent 成本的关键
   - 监控缓存命中率

5. **上下文工程 > Prompt 工程**
   - 精心组装上下文 > 写固定 prompt
   - 压缩优于扩展
