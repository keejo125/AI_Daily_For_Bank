## 什么是 Harness

Harness Engineering 是 Claude Code 的核心架构设计，解决 AI Coding 的核心问题：

- **上下文管理**：如何高效组织海量代码上下文
- **任务编排**：如何让 AI 有条理地完成复杂任务
- **记忆机制**：如何让 AI 记住工作进展

## Claude Code 7层记忆机制

### 层级结构

| 层级 | 名称 | 说明 |
|------|------|------|
| L1 | 工作记忆 | 当前会话的对话历史 |
| L2 | 项目上下文 | 项目文件结构、规范 |
| L3 | 任务历史 | 已完成的子任务记录 |
| L4 | 工具结果 | 执行的命令和结果 |
| L5 | 外部知识 | 检索到的相关代码 |
| L6 | 系统提示 | 系统级指令和约束 |
| L7 | 持久化记忆 | 跨会话的知识积累 |

### 记忆关键

**仅 200 行代码文件竟然是记忆关键！**

Claude Code 的记忆不是靠向量数据库，而是通过精心设计的**上下文压缩和摘要**机制实现。

## 工程启示

1. **不是向量越多越好**：精心的上下文工程 > 海量检索
2. **压缩优于扩展**：主动摘要比无限上下文更有效
3. **层级化记忆**：不同层级的记忆有不同的更新策略

## 与 Qoder 对比

| 维度 | Claude Code | Qoder |
|------|-------------|-------|
| 上下文 | 层级压缩 | 向量检索 |

## 源码泄露分析

分析 4756 个源文件得出的 10 个秘密：

1. **多入口架构** - 支持多种使用场景
2. **流式响应** - 实时反馈执行状态
3. **增量更新** - 只更新变化的部分
4. **安全沙箱** - 代码执行在隔离环境
5. **工具编排** - 灵活的函数调用系统
6. **错误恢复** - 自动尝试替代方案
7. **进度追踪** - 任务分解与状态管理
8. **会话管理** - 跨会话恢复
9. **插件系统** - 可扩展架构
10. **测试覆盖** - 高测试覆盖保证稳定性
