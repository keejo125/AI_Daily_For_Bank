# Skill 宇宙：让 AI 成为你的"分身"

> 来源：微信公众号  
> 收录：2026-04-03

## 核心理念

> AI 不再只是一个辅助工具，更像是你的一个"分身"。它不仅能理解你的需求，还能逐渐复用你的习惯、约定和规范。

## 问题背景

**Vibe Coding 的痛点**：
- AI 生成的代码不完全符合代码风格
- 不符合公司的业务规范
- 每次都从零开始对齐

## 解决方案：Skill 沉淀

### 什么是 Skill

Skill = 代码风格 + 通用逻辑 + 业务约束 的沉淀

**作用**：
- 让 AI 生成代码时天然贴合使用方式
- 不需要每次都从零开始对齐

### 如何构建 Skill

1. **Ask 模式描述需求**
   - 把既有方法设计、代码风格、约定规则告诉 Qoder
   - 让它基于这些信息生成

2. **迭代优化**
   - 发现结果与预期不一致 → 回到 Skill 文件调整
   - 不是一蹴而就，而是持续优化

3. **分类沉淀**
   - 统一的异常处理工具类
   - 特定的查询构建方式
   - 业务约束规则

## 实战案例：Spring Boot 后端

### 两个痛点

| 痛点 | 说明 |
|------|------|
| **查询封装** | JPA 有多种查询方式，AI 不一定按项目规范写 |
| **异常处理** | AI 不使用自定义 Error 方法 |

### 对比效果

| 场景 | 无 Skill | 有 Skill |
|------|----------|----------|
| 查询 | 使用 JPA 方法定义原生 SQL | 使用项目规范的动态查询工具类 |
| 异常 | 使用默认异常 | 使用自定义 Error 方法 |

## 完整工作流

```
需求拆分 → Ask 模式对齐需求 → Agent 模式修改代码 → 代码 Review → 及时提交
```

### 关键原则

1. **上下文控制在 20%-30%**
   - 超过后信息混杂，AI 判断力下降
   - 大需求主动拆解成多个小需求

2. **Review 后立即提交**
   - 代码可用就提交，避免后续误改覆盖
   - 出问题可快速回退到稳定版本

## 自我迭代方法论

### 成功时
- 把有效方法沉淀到个人知识库
- 形成可复用的经验

### 不理想时

**问题归因**：

| 类型 | 例子 | 处理 |
|------|------|------|
| **不可控** | 模型能力限制 | 调整模型或更换工具 |
| **可控** | Prompt 不清晰、业务逻辑漏洞、工具使用不熟练 | 优化 Prompt、重构代码、学习工具 |

## Qoder 的更多可能

- 写剧本、打磨爆款文章
- 构思视频脚本
- 创作 MV（《我是 Qoder》）
- 定义它自己的角色形象

**核心洞察**：它不只是一个"写代码的工具"，更是一个可以参与思考、协同创作的伙伴。
