## 阿里云 Hermes 可观测插件方案

### 背景

**Hermes** 是 Nous Research 打造的自治式 AI Agent 运行框架，能够持续运行、调用工具、积累经验。阿里云为其提供了一套基于 OpenTelemetry 的可观测插件方案。

### 核心解决的问题

| 问题类型 | 具体表现 | 解决方式 |
|----------|----------|----------|
| 过程不可见 | 只能看到输入输出，中间多轮推理/工具执行空白 | 结构化调用链 Trace |
| 成本不可归因 | 知道总 Token 消耗，不知道哪一轮拉高了成本 | 每轮模型调用单独记录 Token |
| 性能不可拆解 | 用户说"变慢了"，无法定位 | TTFT 首字延迟 + 各阶段耗时拆分 |
| 结果不可复盘 | "看起来成功了但结果不对"，路径已偏 | 完整链路还原，从猜原因到看路径 |

### 技术亮点

- **标准协议**：基于 OpenTelemetry，兼容 GenAI 语义约定，不绑定单一云服务
- **Streaming 优化**：单独记录 TTFT（Time To First Token），区分首字慢 vs 整体慢
- **安全审计**：识别越权访问、异常数据导出、恶意提示词注入
- **Metrics + Trace**：既能复盘单次请求，也能观察成本波动趋势

### 对我行的启示

1. **Agent 可观测是生产落地的必要基础设施** —— 不是"有了再用"，而是"用了必须有"
2. **OpenTelemetry 标准值得跟进** —— 避免被单一厂商锁定，保持迁移灵活性
3. **TTFT 指标对用户体验至关重要** —— 智能研发工具的"响应速度"感知主要来自首字延迟

## 阿里云千问云：面向 Agent 的模型网关

### 背景

2026年5月20日，阿里云在云峰会上发布全新AI产品官网"千问云"（qianwenai.com），定位为**面向 Agent 的模型服务网关**。这是继模型服务平台化之后，进一步面向 Agentic 时代做的全面升级。

### 核心能力

- **150+ 主流模型聚合**：Qwen、GLM、Kimi、DeepSeek、Wan、HappyHorse 等，一站选型对比
- **全面 Skill 化和 CLI 化**：将模型选型、调用、认证、用量查询等完整链路封装为 Skills 和 CLI，Agent 可直接调用，无需人工介入
- **多模态自动路由**：Agent 自动识别任务类型，路由到视觉/图片生成/视频生成等专用模型
- **成本管理模式**：Agent 实时拉取用量数据，自动分析趋势、识别异常，提供成本优化建议

### 首批支持的 Agent 工具

OpenClaw、Hermes Agent、Claude Code、**Qoder**（通义灵码）等主流开发工具均已首批集成。

### 开源与商业模式

- Skills 和 CLI 均已在 GitHub 开源
- 两种付费模式：按量付费 + Token Plan（适合高频 AI 编程和 Agent 场景）

### 对我行的启示

1. **模型网关化是趋势**：随着模型种类增多，统一网关替代直连成为必然，可减少集成维护成本
2. **Skill 化降低接入门槛**：我行智能研发平台可借鉴 Skill 封装思路，将常用工具和能力标准化
3. **Qoder 首批集成意义**：我行已在用 Qoder，千问云 Skill 化后可进一步降低调用复杂度
