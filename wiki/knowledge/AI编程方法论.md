# AI 编程方法论

> 整合AI编程核心方法论，从Vibe Coding到Memory系统

---

## 核心共识

### 1. Vibe Coding：新的编程范式
AI 编程演进路径：[1][2]

| 阶段 | 人类角色 | AI角色 | 特点 |
|------|----------|--------|------|
| 1.0 辅助编码 | 编码者+审核者 | 代码补全 | 效率有限 |
| 2.0 Agent模式 | 审批者 | 自主执行 | 效率大幅提升 |
| 3.0 Vibe Coding | 架构师 | 全程实现 | "用嘴巴编程" |

**Karpathy观点**："AI执行+人类审核"成为标准流程，人类不再需要理解代码，只需要描述清晰意图。[1]

### 2. 渐进式 Spec 方法论
模型能力决定方法论选择：[2]

| 层级 | 能力 | 适用方法 |
|------|------|----------|
| T0 | 一次生成全链路 | 直接使用 |
| T1 | 多提示可达接近效果 | 渐进式提示 |
| T1.5 | 基本可用但漏边界 | Spec前置约束 |
| T2 | 只能写骨架 | 人工+AI协作 |

**核心公式**：
```
Agent = while循环 + Tool Use + 工具执行器
```

### 3. Memory 是核心能力差距
AI 的差距不在"理解"，而在"记忆"：[3][4]

| 场景 | 无记忆 | 有记忆 |
|------|--------|--------|
| 多轮对话 | 每次从零开始 | 上下文连贯 |
| 长期协作 | 无法积累 | 越用越懂你 |
| 知识沉淀 | 不复用 | 自动复用 |

## 观点分歧

### 关于"是否需要理解代码"
| 观点 | 来源 | 立场 |
|------|------|------|
| 不用理解代码，相信AI | [1] Vibe Coding | 激进派 |
| 仍需保持批判思维，验证AI输出 | [1] Vibe Coding | 保守派 |

> **整合理解**：Vibe Coding 不代表完全放弃审核，而是将审核焦点从"语法"转向"逻辑"和"业务"。

### 关于模型能力与效率
| 观点 | 来源 |
|------|------|
| 模型能力差距在"一次成功率"和"边界处理" | [2] Spec方法论 |
| AI价值不在能做多少，而在解决真实问题 | [5] AI产品断舍离 |

> **整合理解**：效率提升 ≠ 功能堆砌，"200美元测试法"——用户愿不愿意为此付高价，才是真正的价值判断标准。

## AI 产品设计理念

### 断舍离四原则
来自得到AI产品实践：[5]

| 功能类型 | 常见误区 | 正确做法 |
|----------|----------|----------|
| 模板堆砌 | 功能越多越好 | AI自动匹配，用户无感 |
| 伪需求 | 想象用户需要 | "愿不愿意付200美元？" |
| 信息压缩过度 | 一键生成完整结果 | 保留用户加工空间 |
| 门槛过高 | 追求技术领先 | "我奶奶能不能用？" |

### "200美元测试法"
问团队一个问题：**"如果这个功能标价200美元，你们愿不愿意掏钱？"**

如果摇头 → 不做。

## Memory 架构体系

### Raw Ledger → Views → Policy 闭环
Memory系统三层抽象：[4]

```
Raw Ledger（权威记录）
       ↓
Views（派生视图：向量/KG/Timeline/Skill Index）
       ↓
Policy（控制规则：什么存、怎么用）
       ↓
Commit（回写）
       ↓
Provenance（可回放审计）
```

### System 1 + System 2 分工
| 系统 | 特点 | 负责 |
|------|------|------|
| System 1 | 快、即时、自动化 | 实时上下文、热点检索 |
| System 2 | 慢、深度、周期性 | 知识蒸馏、视图更新 |

### ProcMEM 技能固化三阶段
| 阶段 | 说明 |
|------|------|
| Phase 1 | 发现模式，写入 MEMORY.md |
| Phase 2 | 整理成 SKILL.md |
| Phase 3 | 评估后可发布为标准技能 |

## 核心原则总结

### Vibe Coding 四原则
1. 不要试图理解代码——相信AI
2. 专注于需求描述——清晰的意图 > 精确的指令
3. 接受不完美——快速迭代比完美重要
4. 保持批判思维——验证AI的输出

### Spec 编码七步法
1. 需求澄清
2. 技术约束确认
3. 设计评审
4. AI生成代码
5. 自测验证
6. 代码审查
7. 持续集成

### Memory 最佳实践
- **上下文控制**：20%-30%利用率最优
- **存储策略**：精选 vs 全存，质量 > 数量
- **更新机制**：增量更新，避免覆盖重要旧信息
