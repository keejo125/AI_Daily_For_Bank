# 2026-05-10 工作记录

## 任务：制定公司智能研发规格驱动编程算力分配方案

### 讨论过程与共识

与用户逐项讨论5个核心议题，形成以下共识：

**讨论点一：选人策略**
- 强算力（1,800人高配额）+ 弱算力（8,200+人低配额）使用同一套强模型
- 配额内体验好，配额用完自动降级小模型（效果差但能用）
- 代码补全全员不限，小模型无瓶颈
- 团队优先 + 骨干兼顾的策略

**讨论点二：组织层级与配额方式**
- 方案2：全个人配额（开发简单，按团队聚合评估成效）
- 骨干界定：以规格驱动为主要工作方式
- 评估指标：代码量、Token使用量、入库率
- 标杆团队：自行申报→审批制，总人数≤1,800

**讨论点三：算力池分配**
- 首批：申报审批制，审批时兼顾各基地/部门平衡
- 动态调整：月度评估效果/浪费情况，与1,800以外人员置换
- 团队尽量保持稳定，除非整队效果差
- 个人可灵活置换

**讨论点四：重置周期**
- 两个备选方案：按日重置 / 按半日重置
- 参考：招商银行使用半日重置
- 配额针对规格驱动模式，传统补全不限制

**讨论点五：激励与约束**
- 用完降级小模型 → 天然约束
- 动态置换 → 强激励
- 效果评估绑定配额 → 目标对齐
