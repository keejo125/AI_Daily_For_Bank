## 第一章 调研概述

### 一、调研背景与目的

本报告聚焦2026年4月业界智能研发领域的最新动态，面向工商银行智能研发建设进行对标分析。报告覆盖三大维度——岗位智能体建设、支撑能力、底座能力，调研对象涵盖国际头部厂商、国内头部厂商及银行同业，旨在识别变革性信号与持续性演进，为工行智能研发战略规划提供决策支撑。

### 二、变革性与持续性判断标准

| 分类 | 定义 |
|------|------|
| 🔴 **变革性** | 满足任意一条：①范式转变 ②能力跃迁 ③生态重构 ④首次商用落地 |
| 🟡 **持续性** | 已有能力的质量/速度提升、现有产品功能扩充、已有方案推广扩大、竞品功能跟进 |

### 三、各维度调研结论概览

#### （一）岗位智能体——编码与测试阶段

**国际层面**：🔴 变革性突破为主。Claude Opus 4.7编程能力暴涨（SWE-bench Pro 64.3%），Cursor 3发布（SpaceX收购期权），微软打破OpenAI排他合作（Claude Opus 4.7首日集成），GitHub Copilot转向Token计费，OpenAI GPT-5.5登陆AWS Bedrock突破Azure独家。

**对工行影响**：Cursor删库事件警示工行必须建立严格的AI代码操作安全机制。建议禁止AI工具直接执行删除操作，建立代码变更的多级审批流程。
