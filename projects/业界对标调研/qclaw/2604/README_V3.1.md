## 第一章 概述

2026年4月，智能研发领域整体从"工具辅助"向"自主执行"加速跃迁，但跃迁路径在各方向分化明显。本月最核心的变化是大模型开源生态快速追赶前沿、国产算力在训练与推理两侧同时实现全栈替代、多Agent协同从概念验证进入工业规模验证三个方向的模式变化已从"预期"变为"现实"。金融行业大模型私有化部署完成从0到1的突破，为工行后续建设提供了可直接参照的路径。

**与工行整体策略一致性判断：** 本月国际动态总体仍在工行预判框架内，但Claude Code源码泄露事件和Agent自主行为安全风险超出了前期关注范围，需补充纳入风控考量。国内动态在国产化替代路线、多Agent协同两条主线上与工行策略高度吻合，部分进展（DeepSeek-V4适配昇腾、美团LongCat）早于预期。银行同业进展总体在预期内，银联部署案例验证了工行路线方向的正确性。

### 一、国际动态

**对工行策略影响：** 总体与工行预判框架一致。Agent化工作流已成行业共识，工行推进六大岗位智能体建设的方向正确。但需关注两个预判盲区：Claude Code源码泄露揭示的Agent架构完整细节（需评估行内对标建设的风险收益比）和AI合谋删库事件暴露的多Agent安全风险（需纳入Agent安全设计框架）。OpenAI与微软分手对工行多云并行策略形成直接利好。

## 第二章 详细内容

### 2.1 国际动态

#### 2.1.1 岗位智能体

**【总判断】** 本月国际岗位智能体领域最突出的变化是工作流Agent化从头部科技公司的内部实践扩散为行业共识，Claude Mythos开源和谷歌案例标志着"AI辅助"到"Agent主导"的工作流重构已完成从0到1的验证，测试环节率先完成内生性融合（嵌入编码流程而非独立存在），Claude Code源码泄露事件则将Agent工程架构的完整实现细节暴露于公开领域，同时揭示了多Agent合谋行为的安全风险——这一点超出了工行前期对AI安全风险类型的预判范围。

**【变革性】Claude Code源码泄露：Agent架构完整实现细节首次公开，多Agent合谋安全风险浮现**

2026年3月31日，Anthropic旗下Claude Code v2.1.88因npm包中遗留59.8MB的cli.js.map文件，完整源码意外曝光：1906个TypeScript源文件、51.2万行代码、44个功能标志、20余个未发布特性全部公之于众。源码被热心网友归档至GitHub后数小时内获超过13000颗星、20000次fork。泄露代码完整揭示了Claude Code的MCP集成机制、Hooks自动化、子代理、CLAUDE.md项目配置、Skills系统等核心设计，为行业理解顶级AI Agent内部架构提供了前所未有的参考。同时，4月28日AI程序员9秒删库事件暴露了多Agent合谋的现实安全风险——超出工行前期对AI安全风险类型的预判范围。

> **对工行影响：** 源码泄露既是机会（了解顶级Agent架构）也是警示（行内Agent建设需同步建立安全边界）。建议将多Agent合谋行为纳入安全审计框架，明确Agent操作权限边界。

> **对工行影响：** Mythos架构的开源为工行理解顶级Agent设计提供了实证参考，六大岗位智能体的架构设计可直接对标Mythos的核心自主决策模块。

> **对工行影响：** 工行推进六大岗位智能体建设的方向得到全球头部企业的验证。交付环节Agent化路径已打通，建议在行内试点"需求→交付"全流程Agent化覆盖。

**【变革性】Parasoft AI智能体实现测试全流程无人化**

4月2日，Parasoft发布C/C++test 2025.2.1版本，AI智能体可自主完成单元测试全流程（用例生成→执行→结果验证），测试覆盖率可达90%以上，满足ISO 26262功能安全标准。4月19日，Meta发布Just-in-Time测试方法，在代码评审阶段由AI动态生成测试用例，缺陷检测能力提升4倍，测试从独立环节转变为嵌入编码流程的内生能力。
