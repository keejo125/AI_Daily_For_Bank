4、Agent Workspace：夯爆了

整体体验下来，最喜欢的功能是办公室的交互设计——纯白调的 3D 空间，几个工位、一台跑步机、一个咖啡角。每个 Agent 都是一匹小黑马：Marvis 坐在主工位上盯着屏幕，App Agent 在跑步机上跑着，File Agent 蹲在马桶上处理文件，Computer Agent 戴条绿松石围巾在打王者荣耀。谁在干活、谁摸鱼，一眼看穿。

最戳人的是空闲状态。没任务的时候，Agent 不是消失，不是转圈，而是工位空着，仿佛只是去接了杯咖啡。偶尔两匹马还会互相递气泡：「到你了」。你知道它们没崩、没卡、没偷偷执行什么，就是单纯地在等你一句话。

状态透明不是锦上添花。多 Agent 协同最怕黑盒——你不知道谁在干嘛、为什么卡住、会不会越权。这间办公室把协作链路摊在桌面上：可看、可盯、可随时介入。

9、最期待的功能-手机端连接：还要等等

【手机端连接现状】

我是苹果手机用户，所以微信 bot 和手机端是我最期待的功能——期待到每次打开设置面板都会多看一眼那个「iOS 端敬请期待」的标注。

微信 bot 绑定后，手机就是电脑的远程终端。人不在电脑前，对着微信发一条消息，Marvis 收到、执行、回结果。本质就是个异步指令通道，不推画面、不镜像屏幕，只传结果。轻量够用，但能力有限——所有需要「看着屏幕确认」的操作，手机端都做不了。

10、结尾

测完一圈回头看，Marvis 的定位在我心里清晰了。

2

026 年初，桌面 Agent 赛道彻底卷起来了，OpenClaw 走开源极客路线，能力全开但风险自担——GitHub 25 万星标、一周 200 万访客，也带出了「龙虾悖论」：你让它做的越多，给的权限越大，安全风险就越高。

各家都在抢「通用桌面 Agent 入口」。

Marvis 选了一条不同的路：

安全实用主义。

它不是最激进的，不是最开放的，但它可能是现阶段最适合普通用户的

。

六个Agent 出厂预置、零配置上手，不需要用户自己搭工作流、找 Skill、装插件——这是腾讯应用宝团队在 PC 和移动端底层生态里泡了十四年才攒出来的工程底座。效率模式和隐私模式双轨并行：日常任务端云协同，速度快；敏感场景端侧模型处理，数据完全不出电脑。L2 级安全兜底——删除、修改、调整核心配置，先给执行计划，强制确认；支付等更高敏感动作，必须用户本人完成。

这套能力的本质是 harness——不是让 AI 无限制地跑，而是给它套上缰绳

，让它在安全边界内全力冲刺。对普通用户来说，这比「能力上限」更有价值。你知道它在干什么，知道怎么介入，知道数据不会乱跑。

产品可用性就建在这套 harness 之上：

多 Agent 状态可视化、子对话折叠、权限确认卡片，每一个细节都在降低信任成本。

市场上竞品要么跑得太快忘了安全，要么守得太严丢了体验，Marvis 在中间找到了一个暂时说得过去的平衡点。

更让我意外的是团队节奏

。今天是周

末

，凌晨还在写这篇测评的时候，打开 Marvis 发现版本号又跳了一格——它自己在后台悄悄更新了。要知道这产品 5 月 20 号才正式上线，四天时间里，反馈的 bug 被修了几个，Skill 调用逻辑也有优化痕迹。应用宝团队这种周末还在迭代的劲儿，让我更加对产品的未来充满兴趣。

最后说点感受。作为一个每天追 AI 产品的人，我已经很久没有对什么东西感到兴奋了。不是产品不够多——恰恰相反，太多了。每周都有新框架、新模型、新概念，多得让人麻木。

但 Marvis 是少数几个让我重新兴奋起来的——不是因为它技术多前沿，而是因为它把前沿技术做成了一个普通用户能直接上手、敢放心用的东西。

它验证了一个方向：AI 助手正在从「对话工具」变成「执行系统」。未来的竞争不再是「谁更会聊天」，而是「谁能更安全、更可靠地帮你干活」。Marvis 的出现，意味着操作系统层级的 AI 不再是远景，而是一个你今天就能装、今天就敢用的产品。

它不完美，但

它是

真实可用的。对普通用户来说，现在就可以用；对技术用户来说，值得持续关注它的迭代。

毕竟，你需要的不是最炫的功能，而是最可靠的伙伴。Marvis 还在路上，但方向是对的。这就够了。
