过去几个月，Coding Plan 让大批开发者尝到了"模型包月"的甜头。阿里云百炼、智谱、Kimi、MiniMax 先后推出包月方案，开发者按月订阅后，把 API Key 配进编程工具，就能在 IDE 里直接调用这些国产大模型。Coding Plan 第一次让"用大模型写代码"从尝鲜变成了日常。

但开发者需要的不只是 Coding Plan 的升级版，而是一个覆盖多模态、计费更透明、能在企业内规模化使用的模型订阅方案。

今天，

灵码正式支持 Token Plan 团队版

——由阿里云百炼推出的 Coding Plan 全面升级版。多模态、多模型，按 Credits 统一抵扣。

01

丰富的模型支持

把 Token Plan 的专属 API Key 配到灵码中，就能直接使用 qwen3.6-plus、deepseek-v3.2、glm-5、MiniMax-M2.5 等多款 SOTA 模型。

更多模型将在未来持续上线更新，大家敬请期待。

02

真实的能力升级

Token Plan 不是 Coding Plan 换了个名字。它在多个层面做了根本性的改变与升级：

多模态覆盖：

不只是文本生成。Token Plan 同时支持深度思考、视觉理解、图像生成（qwen-image-2.0 系列、wan2.7-image 系列），语音识别和语音合成也将于未来陆续上线。从编码到日常 AI 办公，一个订阅全部搞定。

企业席位管理：

提供标准版、高级版、尊享版多档位座席套餐，匹配不同角色使用强度，自由搭配。

数据安全承诺：

默认不使用用户数据进行模型训练，满足企业级数据隐私要求。

03

轻松配置

1、前往阿里云百炼控制台，订阅 Token Plan 团队版，获取专属 API Key

。

Token Plan 和模型 API 属于完全不同的产品，Token Plan 不能抵扣模型 API 的用量，需要独立区分出用户的调用。产品采用专属的 API Key 和 专属的 BaseURL

2、下载或更新灵码到最新版，进入

Lingma设置->模型

。

3、选择添加模型后，粘贴专属 API Key

。

配置完成后，Token Plan 模型出现在灵码的模型列表中，和内置模型一样使用。

Token Plan 团队版当前已开放单座席购买。未来将开放多席位购买、企业级席位管理和超额按量付费功能。
