# Snowflake 让 Builder 从想法走向生产

> 原文链接：https://mp.weixin.qq.com/s/Gop3mMP4eUkDygXOrApdvA
> 公众号：InfoQ

4 images

这场高信息密度的 Builder Keynote，直击企业 AI 的真实构建现场：开发者、数据工程师、分析师与业务团队，如何在同一套可信数据与治理边界内，将各种碎片化的想法快速转化为应用、流程和业务结果。

Snowflake 分析主管 Carl Perry 指出，当今创新的最大障碍已不再是时间和工程资源，而是无处不在的碎片化（多云、多格式、多工具和工作流）。Builder 的核心时间，不应再虚耗在“手动拼装系统”上。

为此，Snowflake 正在驱动三大核心转变：

🌐 数据构建：工作走向数据

数据无需搬迁，工作可直接来到数据身边。例如 NYSE / ICE 在管理 40 万亿美元市值、12PB 数据及月均 2 亿+ 次查询的高监管环境中，依然能在 Snowflake 统一的既有治理边界内，安全、高效地进行 AI 自助分析。

✨ 让 AI 参与构建：从代码助手到 Agent 团队

Snowflake 的 AI 助手 CoCo 已进化为全生命周期的协同者，深度参与规划、拆解、生成与测试。在 Data Donuts 演示中，CoCo 仅凭一条 Prompt 便连接了 Iceberg、GCS、Databricks 等 30 多张跨平台业务表，完成情绪分析与流失预测。 更具突破性的是，CoCo 展现了 Agent 团队的自主协作：Planning、Implementer、Tester 和 Reviewer 各司其职，在代码生成前进行推演与审查，提前规避并发冲突（Race Condition）和 API 调用错误。

🚀 洞察走向行动：AI 驱动业务闭环

Demo 的终点不是一份静态报告，而是 Slack 中触发的 P0 级实时告警：精准识别出 92% 的流失置信度（Churn Score 71→100），并直接推送 24 小时、3 天、7 天的挽回行动方案。AI 不仅在发现风险，更在将原因、优先级和下一步动作直接交付给业务一线。

Snowflake 的核心愿景，是让每一位 Builder 都能在安全可信的环境中，将想法变应用，将洞察入流程，把 AI 转化为确定性的业务成果。

Builders power the agentic enterprise.

Snowflake powers the builders.❄️
