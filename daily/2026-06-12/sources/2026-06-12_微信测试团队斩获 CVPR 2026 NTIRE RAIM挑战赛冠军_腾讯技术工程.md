导语

在 CVPR 2026 NTIRE "The 3rd Restore Any Image Model (RAIM): Professional Image Quality Assessment - Track 1" 挑战赛中，来自 微信测试中心的 IH-VQA 团队斩获大赛冠军。本届 RAIM 由 OPPO Y-Lab 与香港理工大学 VC-Lab 联合主办，赛题首次将"专业级图像质量评估"从单一标量打分推进到 "高分辨率多模态推理 + 专家级偏好预测" 的全新范式。比赛中，IH-VQA 团队首创的 iDiff 可解释成对差异感知框架（Interpretable Difference-aware Framework for Pairwise IQA），在 ACC 偏好准确率与思维链推理质量两大维度均刷新了赛事基线，最终以 Final Score 0.7305 在高校与业界的众多强队中夺魁。

本次比赛获得亚军的是 VCIP Pi Group（0.7253），季军是 I² Group（0.7040），4-6 名分别为 fugui、LZ、ongaku 团队（中科大、国科大、西工大等一众高校实验室团队）。在去年成功夺得 CVPR 2025 NTIRE Image-Text Alignment 冠军、ICCV 2025 MIPI Detailed IQA 冠军后，微信测试中心IH-VQA算法团队再次站上国际顶级视觉质量评估赛事的最高领奖台，进一步巩固了其在 AI 多模态质量评估领域的领先地位。

一、问题与挑战：为什么单纯的"画质分"已经不够用

在视频号业务、实时音视频通话、直播、朋友圈以及其他和画质强相关的业务场景中，画质评估一直是一个被反复讨论的问题。过去十多年里，IQA 的主流范式是回归一个标量分数——MOS、PSNR、SSIM、或近年的 iQA  / SAMA / Q-Align。这套范式解决的是"A 和 B 谁分高"的问题。

但在我们日常的工程实践中，业务侧问得更多的其实是另一类问题：

⦁ 新版本编解码器和旧版本相比，哪些场景退化了，退化在哪？

⦁ 同一张照片，A 设备和 B 设备的成片差异，到底差在肤质、噪点、还是动态范围？

⦁ AIGC 修图模型出来的几张候选图，为什么人工觉得这张更好、那张不行？

这些问题的共同点是：业务侧不只要一个分数，而要一段可对照、可归因、可执行的解释。标量打分在这里就显得乏力——

它告诉你 A 比 B 高 0.07，但不能告诉你这 0.07 来自哪里。

随着 MLLM 能力上升，"让模型像专家一样讲理由"开始有了工程上的可行性。但在专业摄影质量评估这个具体场景下，直接拿现成的

MLLM

来用，会遇到两个具体的坎：

高分辨率细节会在前端就丢掉。专业摄影图的失真线索（纹理、噪点、伪影）是像素级的；而通用 ViT/CLIP 编码器为了配合 token 预算会把图 resize 到很低分辨率，关键信息直接被压没了。

通用

MLLM

容易给出"听起来对但说不到点上"的描述。它会说"图像 A 看起来更清晰、更自然"，但不会按影像工程语言去做"肤质纹理 vs 修复伪影"、"高光留白 vs 动态范围"这样的细粒度对比。

这两个坎是这次工作的真正起点。在我们研究此问题的过程中，发现学术界也有相关同仁在视觉顶会CVPR 2026 NTIRE WORKSHOP中发布了类似的工作探讨，我们有幸参与其中，并在最终的角逐中获得了冠军。本文旨在记录我们针对这个业界难题的方法设计、消融验证与业务落地的一系列工作。

二、赛事任务定义与现有方法的局限

2.1

赛事数据集

本次比赛由 OPPO Y-Lab 与香港理工 VC-Lab 共同构建发布，专注于专业摄影场景——涵盖

人像（Portraits）与风景（Scenery）

两大主类。每条样本由一对来自不同设备或不同设置下拍摄的高分辨率图像（Image A 与 Image B）构成，并由专业摄影师/影像专家给出：

● 专家偏好 (Expert Preference)：A 或 B 的偏好二选一答案；

● 推理段落 (Reasoning Paragraph)：解释偏好理由的详细推理段落，覆盖纹理、真实感、噪声、动态范围、伪影等专业维度。

数据集样本示例

2.2

赛事评价规则

赛题采用 "偏好准确率 + 推理质量" 双口径评价，最终得分由两部分加权融合：

⦁Preference Accuracy（ACC）：模型预测专家偏好（A/B）的二分类准确率，衡量"看得对不对"；

⦁Reasoning Quality（推理质量）：评估推理段落的质量与深度，采用混合方法——

a.语义相似度：BLEU-4 + ROUGE-L，衡量与专家原文的覆盖度；

b.LLM Judge：高能力裁判 LLM 从逻辑连贯性、幻觉控制、图像内容对齐三维度打分。

c.最终得分：Final Score = 0.5 × Phase 2 Score + 0.5 × Phase 3 Score（Phase 3 由 LLM Judge 介入，权重较高）。

评价指标关系

2.3 比赛阶段

本次比赛分为三个阶段：

⦁Phase 1（2026.01.26 – 2026.02.08）：训练数据发布，参赛者自由调优；

⦁Phase 2（2026.02.08 – 2026.03.11）：验证集开放，在线提交结果；

⦁Phase 3 / Final Evaluation（2026.03.11 – 2026.03.18）：测试集隐藏，参赛者提交模型与推理代码，组委会内部运行评测，LLM Judge 介入评分。

2.4 现有方法的局限性

现有图像质量评估方法多侧重整体打分或简单优劣判断，难以解释质量差异的具体来源。在真实业务场景中，两张图像往往仅在局部清晰度、纹理保真度、噪声、色彩还原或人物细节上存在细微差别，传统方法既难以准确定位，也无法给出符合人类主观感知的解释。

此外，现有方法在高分辨率图像、复杂场景与局部退化问题上仍存在不足：单一模型容易忽略局部关键损伤；仅依赖通用视觉特征或文本推理的模型，则可能判断不稳定、解释不充分，甚至与图像内容不一致。因此，如何同时实现准确的质量比较、细粒度问题定位与可解释的质量分析，仍是图像质量评估领域的重要挑战。

三、我们的方案：iDiff——一个可解释的“差异感知”双分支框架

面对 RAIM 赛事"既要判得准、又要说得清"的双重考验，传统 IQA 模型很难在同一套参数下兼顾鲁棒的判别能力与高质量的语言推理：判别式模型擅长"打分"却不擅长"开口讲";生成式 MLLM 能"开口"却不一定"判得稳"。为此，团队从问题本质出发，提出 iDiff（Interpretable Difference-aware Framework）——一个面向成对图像质量评估的可解释差异感知框架：用两个互补的分支分别承担判别决策与推理生成，让"做判断"与"讲清楚"各司其职、互不掣肘，最后再以协同的方式融合输出。本章将围绕设计动机、核心模块与最终效果展开，详细介绍 iDiff 的整体思路、关键技术与实现细节。

3.1 双分支设计的动机

回顾近年来主流的图像质量评估范式，现有 IQA 方法大致可以分为两类，它们各自在自己擅长的赛道上表现优异，但都难以同时胜任 RAIM 任务对"准确判别"与"可解释推理"提出的双重要求：

⦁判别式回归模型（如 MUSIQ、MANIQA、SAMA）：直接面向打分目标进行优化，在感知一致性和数值预测上较为稳健，但缺乏天然的语言出口，难以输出可被专家阅读的解释，"知其然而不能言其所以然"；

⦁MLLM 质量评估器（如 Q-Align、DepictQA、Co-Instruct）：天生具备生成自然语言解释的能力，但在细粒度成对比较场景中，生成式预测的稳定性较弱，容易出现幻觉、自相矛盾或前后不一致。

由此我们提出了 iDiff 的核心论点：

"准确的偏好预测"与"高质量的推理生成"应当通过互补方式建模，而不应由单一分支独自承担。

基于这一论点，iDiff 被设计为双分支协同架构：Answer Model 专注于鲁棒的判别式偏好预测，Thinking Model 专注于结构化的专家级推理生成。两个分支在训练阶段相互独立优化，避免相互干扰；在推理阶段又通过 Answer-aware 监督的方式实现耦合，让推理过程围绕已经稳定的判别结论展开。这种"先分而治之、再协同闭环"的设计，最终既保证了"答案"的稳定性，又保证了"过程"的可读性，与赛事评价中 ACC + 推理质量的双口径目标天然对齐。

3.2 Answer Model：判别式偏好预测分支

Answer Model 的目标非常明确——给定一对图像 (I_A, I_B)，直接判别哪一张更优。围绕这一目标，团队从输入表征、内容建模与模型融合三个层面进行了针对性设计，由以下三项关键模块组成。

Answer Model整体设计图（论文 Figure 2）

3.2.1 多视图输入分解（Multi-view Input Decomposition）

赛事原始数据将左/右候选图打包为单张拼接图，每个样本同时包含全局视图与裁剪局部视图。直接使用拼接表征会让模型难以显式建模"左 vs 右"的相对差异，相当于把两张图"糊"在一起喂给网络，模型只能依赖隐式特征区分。团队因此没有沿用拼接表征，而是将每个样本显式拆解为 4 个对齐输入：

其中

为左/右全局视图，

为对应的裁剪局部视图。该分解使模型能够分别捕捉全局感知一致性（构图、曝光、整体自然度）与局部失真细节（纹理、噪声、伪影)，二者对成对 IQA 同等关键，缺一不可。消融实验也表明，仅这一项设计就能将整体准确率从 0.5784 提升到 0.6961，是后续模块增益的基础；并且这一收益在 5 种不同骨干上一致出现，说明 Split L/R 是一种与架构无关的鲁棒改进。

3.2.2 内容感知模型专门化（Content-aware Model Specialization）

团队进一步观察到，不同语义内容下的质量线索存在显著差异，把所有数据混在一起训练会带来明显的类内异质性，模型容易"顾此失彼"：

⦁人像（Portraits）：以面部纹理、肤质平滑度、发丝边界、修复伪影为主导；

⦁场景（Scenery）：以建筑结构、植被、文字清晰度、整体自然度为主导。

为此，团队将训练数据划分为 Portraits / Scenery 两个子集，并分别训练独立的 Answer Model，显式让每个模型只关注一个语义域内的关键失真模式，避免特征学习的目标互相稀释。这一专门化策略有效降低了类内异质性、提升了对领域相关失真的敏感度，整体准确率在前一阶段基础上进一步提升到 0.7745，其中场景子集的准确率从

0.6154 跃升至 0.8077

，提升尤为显著，验证了"按内容分而治之"在小样本 IQA 场景下的有效性。

3.2.3 多骨干集成（Multi-backbone Ensemble）

不同骨干网络在建模纹理、边缘锐度、伪影模式上各具互补的归纳偏置：CNN 系骨干对局部纹理与边缘更敏感，Transformer 系骨干在全局一致性与长距离依赖上更具优势，混合架构（如 MaxViT）则在两者之间取得平衡。团队在 5 个骨干上分别训练并融合：WideResNet-50-2 · EfficientNet-B2 · ConvNeXt-Small · Swin Transformer V2-S · MaxViT-T；最终 Answer Model 的预测通过跨骨干等权投票融合得到。实验显示，等权投票相比任一单一模型显著提升泛化能力（整体准确率由

0.7745 进一步提升至 0.9118），且效果优于按验证集精度加权的版本，说明在小样本成对 IQA 场景下，等权融合往往比"过拟合验证集的加权"更稳健——这也是团队最终选择简单等权策略的关键原因。

3.3 Thinking Model：结构化推理生成分支

Thinking Model 在 Qwen3-VL-8B-Instruct 基础上进行 LoRA SFT，并采用渐进式增强策略——从一个最朴素的"图像对 → 解释"基线出发，逐步引入模板、特征与答案监督，每一步都解决前一步留下的具体问题，引导 MLLM 完成结构化、可验证、与专家风格高度对齐的成对推理。值得一提的是，团队在 MiniCPM-V-4.5、GLM-4.6V-Flash、Ovis2.5-9B、InternVL3.5-8B 等多个 MLLM 上验证了该策略的通用性，结论一致——

渐进式增强不是某个特定模型的专属技巧，而是一套可迁移的范式

。

Thinking Model策略演变图（论文 Figure 3）

3.3.1 结构化推理监督（Structured Reasoning Supervision）

模型不直接预测最终标签，而是要求按以下结构化格式生成中间推理：

<thinking> ... </thinking>

<answer> A / B </answer>

推理被进一步分解为纹理、噪声、锐度、自然度等多个感知维度的细粒度对比，再加一个全局结论。这种"先讲依据、再下判断"的 rationale-first 范式鼓励模型进行多属性分析，同时强制保持局部观察与整体判断之间的一致性，避免出现"理由说了一大堆，结论却南辕北辙"的解释幻觉，也让生成结果在结构上天然贴合专家评审的表达习惯。

3.3.2 模板化推理正则（Template-based Reasoning Regularization）

为了进一步稳定输出结构、降低生成方差，团队引入了专家风格的推理模板：每条模板要求生成 4–7 行简洁对比 + 一行强制性的全局总结，每行聚焦"一个区域 + 一个指标"，让推理像专业摄影评测一样可读、可比、可核验。同时按语义域分别设计：

⦁Portraits 模板：强调面部细节，如肤质纹理、发丝边缘、衣物区域；

⦁Scenery 模板：聚焦背景纹理、建筑结构、植被、边缘过渡。

这种领域感知的模板设计显著降低了推理输出的方差，使生成结果与专家风格的 IQA 分析在表达粒度和评论侧重点上自然对齐。消融实验显示，仅这一项就将 BLEU-4 从 0.0221 大幅提升到 0.0858、ROUGE-L 从 0.2200 提升到 0.2825，是 Thinking Model 中单项增益最大的模块；进一步地，领域专属模板也优于通用模板（BLEU-4 0.0801 → 0.0858），印证了"内容感知专门化"思想在生成端同样成立。

3.3.3 多源特征注入（Multi-source Feature Injection）

仅靠视觉信号，MLLM 在面对极相似的图像对时仍容易"看不出差别"，凭直觉给出模糊判断。为此，团队在文本输入中显式注入两类质量相关特征，作为可量化的辅助证据：

⦁传统 CV 统计量：Laplacian、Tenengrad、噪声标准差、高频比、边缘密度、熵、曝光相关比、色彩度、平均亮度；

⦁学习型 IQA 指标：LIQE、Q-Align、SAMA。

这些特征按"图像对 × 裁剪粒度"格式化后追加到 User Prompt，作为结构化辅助证据。这种 Feature-aware Prompting 让模型推理同时锚定在视觉观察与数值测量上，对细微画质差异更敏感，也降低了"凭直觉乱猜"的概率，在更难的细粒度对比中尤其有效。两类特征也具有天然互补性：传统 CV 统计量提供低层、可解释的物理量，学习型 IQA 指标提供高层、与人类感知对齐的语义打分，二者结合让推理更加稳健。

3.3.4 Answer-aware 指令微调（Answer-aware Instruction Tuning）

最后，团队引入了 Answer-aware 监督作为最终的"理由精修"阶段：把 Answer Model 给出的目标答案显式写入 Prompt（例如 ⁠Ground truth: Left is better.⁠ ），模型不再重复预测 Answer Token，而只生成支撑该结论的推理过程。这一设计有两层意义：一方面避免了模型走捷径式的"答案复制"，把生成容量从"猜答案"中解放出来；另一方面强迫模型学习更强的"证据 → 结论"对齐关系，确保理由真正服务于结论而非自说自话。本质上，这一阶段相当于 Rationale Refinement——在判别能力已被 Answer Model 兜底的前提下，让 Thinking Model 把全部容量都用于把"理由"讲得更连贯、更与 Label 一致，让双分支真正实现"判别归判别、解释归解释"的协同闭环，也使 iDiff 在 ACC 与推理质量两端同时拿到稳定增益。

iDiff 不仅在赛事上验证了技术领先性，更在业务场景中展现出独特价值。"成对偏好 + 可解释推理"这两个能力，恰好对应了我们日常质量评估中最高频的三个需求场景：

5.1 视频号双栖盲测：从 VDiff 到 iDiff 的升级

视频号 vs 竞品的双栖画质对比，视频号质量评价 是 IH-VQA 算法落地的核心业务之一。这条线在过去两年经历了从「人工外包标注」「视觉小模型质检」→「iDiff 可解释大模型复核」的三阶段演进。iDiff算法的引入，不仅可以验证单独承担监控任务的可行性；也可以长期基于推理段落统计差异维度分布，让画质优化方向数据驱动化。

5.2 编解码器与算法 A/B 升级评估

每次编解码器或画质算法（超分、降噪、画质增强等）版本迭代，都要回答同一个问题：新版相比旧版，画质到底有没有变好、好在哪里、坏在哪里？ 传统方案——人工外包评测周期长、单刺激 NR-IQA 在小幅相对变化上失真——都难以给出"专业级、可定位"的差异说明。iDiff 替换通用大模型也将是节约成本和提升效果的必然路径。

5.3 一线创作工具与算法选型支撑

视频号一线创作工具线（拍摄美颜、滤镜、AI 修图、AI 海报、AIGC 特效等）每个季度都要做"多候选 PK"——传统做法是把候选效果图发到群里"看图说话"，效率低、口径散、留不下复盘材料。iDiff 算法的引入，可以将这个任务完善为可批量、可解释、可归档的标准流程。
