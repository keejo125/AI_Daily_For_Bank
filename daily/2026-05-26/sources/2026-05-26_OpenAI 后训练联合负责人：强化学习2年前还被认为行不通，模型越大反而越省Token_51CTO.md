聊到模型的幻觉现象，他说，像“承认不知道”、“应对定义模糊的任务”等横向能力的缺失，才是导致大模型在各个垂直领域产生幻觉的根源。而横向能力也是模型能够泛化的关键。

他还讲到了一个反常识观点：模型越大反而推理越高效。这是因为大模型可以通过更强大的权重参数在底层直接“思考”，从而减少在推理时生成的思考Token数量。

AI训练三阶段：预训练是学习一切，中期训练会加大高质量数据权重，后训练是让模型对人类真正有用

Yan:

中期训练，顾名思义，就是处于预训练和整个流程中的后训练部分之间的一个阶段。它的核心理念是，

如果你拥有高质量的数据，而这些数据更能代表你最终模型所希望呈现的效果，你就应该在这些数据上进行过训练（overtrain）

在预训练阶段，我们什么都训练；而在中期训练阶段，我们基本上会加大这类高质量数据的权重

，因为我们认为它们对于训练最终模型更有用。虽然我不能透露我们内部的具体情况，但这绝对是目前整个学术界正在发生的事情，所有的开源模型现在也都有中期训练这个阶段。

后训练最佳策略：先监督微调“克隆”人类行为，再强化学习突破上限

Matt:

监督微调，对。

Yan:

监督微调

“行为克隆（behavior cloning）”

。但

这种方法的问题在于，你永远无法超越你的地面标准（ground truth，即人类提供的标准答案）给你的上限。

而且人类在很多方面其实是挺有局限性的，所以你永远无法超越和你合作的那些人类标注员。而

强化学习（reinforcement learning）阶段则是从行为克隆走向真正的奖励优化

。它的思路是：

我不知道绝对的标准答案是什么，我也不知道完美的回答是什么，但我可以给出一套标准来判断这个回答是对是错，以及这个回答里应该包含哪些要素。

接下来你要做的就是开始优化。你让模型尝试去获得更多的奖励，也就是去优化这个我们称之为“奖励函数（reward function）”的东西。这就超越了你目前所拥有的、或者说人类（至少是与你合作的人类）所能做到的水平。所以我会说这就是两个最主要的阶段。

“可验证奖励（verifiable rewards）”的强化学习

极其容易判断某个结果是对是错，你完全可以为此设置一个二元奖励（binary reward），这可以追溯到我们之前讨论过的 o1 和 o1 preview

。另一种是

没有可验证奖励的强化学习

，在这种情况下，我可能会做

两两比较（pair-wise comparisons），我可以指着说这个答案比另一个答案好，但我并不知道、也无法明确说这就是完美的答案

在开源世界中，人们通常的做法是先进行 SFT，克隆你从网上或人类那里收集到的行为；一旦模型达到相当不错的水平后，他们就会通过强化学习来突破现有的极限。因为如果你直接从强化学习开始，效率会非常低。

强化学习的问题在于，你基本上必须误打误撞地碰到正确答案。

因为强化学习的运作机制是，你从正在训练的模型中进行多次采样，然后说这个是对的、那个是错的，并告诉模型“多生成一些对的那个”。所以你首先必须能碰巧撞上正确的解法。因此，

最好的策略是先尽一切可能接近你能做到的最好水平（也就是行为克隆），然后再进行强化学习。

我之前好几次听说强化学习非常难以捉摸且难以扩大规模，我们整个行业在早期大语言模型的进展曲线中，没有把强化学习作为初始部分的一部分，恰恰就是因为让它运转起来太困难了。扩大强化学习规模的难点究竟在哪里？是数据集的问题，还是不知道奖励在哪里？是这些原因还是其他什么因素？

我会说，

直到两年前，在学术界和研究界中那些不从事强化学习的人，大概率都认为强化学习根本行不通，太难以捉摸了，没法用。

我以前就是这种人。实际上当看到 ChatGPT 问世时（我当时还不在 OpenAI），我看到博客里说他们使用了强化学习，我的第一反应是：“不用强化学习我也能做出一样的东西，因为这只是一种过度复杂的方法。”这恰恰也就是我们做 Alpaca 时开始研究的问题，让我们尝试纯粹通过 SFT、通过这种行为克隆来复现它。是的，而且比如 Yann LeCun 曾有一个著名的比喻，说强化学习就像是蛋糕上的樱桃。所以我认为这确实是大多数人当时抱有的直觉。后来似乎在跨越了某种模型规模之后，也就是

当模型基本上了解了世界的一切、拥有了我们所说的良好的世界先验知识（good priors）之后，强化学习突然就开始起作用了。

而且这不仅仅发生在 LLM 领域，机器人领域似乎也正在进入相同的阶段，他们意识到过去这确实非常难以捉摸，但现在我们使用了本就了解世界万物的模型，它实际上学得相当好。

那么为了回答你关于强化学习现在还有什么复杂之处的问题，首先是基础设施（infra）层面，也就是整体系统的问题。在强化学习中，

从宏观层面上看，正如我所说，你基本上必须对海量的回答进行采样，然后判断哪些是对的、哪些是错的。

这种采样本身就极其昂贵，而且你必须大规模地去进行。另一个问题，也是目前开源世界中大家正在面临的，就是

当我们训练更具 Agent 特性（agentic）的系统时，你只有在漫长的执行流程（roll out）结束时才知道自己是否正确

。因此，

在每个 Token 的层面上，你很难获得关于自己对错的信息。你很难去进行基本的“归因（attribution）”，很难说在你整个回答中，究竟是哪一部分导致了最终的正确

。这更多是机器学习算法层面的一个难题。机器学习最理想的境界是，我能明确告诉你“这个地方做得很好，以后多这么做”。而

这些 Agent 系统以及 Agent 系统的强化学习所面临的问题就在于，在你走到终点之前，你根本不知道哪一部分是好的，哪一部分是不好的

。这是强化学习的另一个重大问题。

Matt:

目前强化学习的前沿是什么？现在似乎充斥着各种缩写术语，比如 GRPO以及其他技术。你们在用什么？你对什么感到兴奋？你认为什么是有前景的？

Yan:

同样的，我不能透露我们在用什么，但例如

在开源世界中，GRPO 似乎表现得非常好

。人们以前用过各种不同的方法，比如 PPO 和 DPO，但大家现在似乎都真正收敛到了这一种方法上。它与其他方法最大的不同在于，你再次回到了我刚才提到的那种简单方法：对尽可能多的回答进行采样，然后判断哪一个是正确的。所以从某种意义上说，GRPO 是一种非常简化的方法。而通常在机器学习中，我们一再看到，那种能够通过算力不断扩大规模的最简单的方法，往往最终效果最好。至少在你提到的那些充满挑战的开源世界中，目前正发生着这样的情况。

（

注：

GRPO（Group Relative Policy Optimization）

：PPO的改进版，通过对同一提示生成多个回答并进行组内相对奖励归一化，实现高效且稳定的推理强化学习。

构建 AI 模型首先是手艺，然后才是科学

Matt:

我脑海中突然闪过一个问题。你知道，大家经常听说 AI 系统不是被构建出来的，而是被“培养（grown）”出来的。在你的日常工作中，你会如何定性它？哪一部分属于科学，哪一部分属于一门手艺（或者说尝试一堆东西然后把最管用的留下来）？

Yan:

是的，这是一个很好的问题。我认为它通常的运作方式是：起初它是一门手艺。大家只是尝试很多东西，并开始建立一套关于什么行得通、什么行不通的心智模型（mental model）。随着时间的推移，我们从这个“手艺阶段”迈向了更具科学性的阶段。科学的、或者说更具科学性的方法，极少是第一批最终派上用场的。这很难，非常罕见的是你采取了一种极其科学的方法，然后说“这就是最完美的方案”，接着你照做它就成功了。这里面其实带有一种“炼金术”的色彩。

人们往往只是对某种东西有一种很好的直觉，然后把它做成了，接着其他人或者这个人自己，开始通过非常科学的手段去尝试改进我们正在做的事情。

我想说，这在机器学习中屡见不鲜。所以，

首先是手艺，然后才是科学，两者都极其重要，只是处于流程的不同阶段

。在工程层面，这绝对是始终必不可少的东西。所以我会说，大多数研究人员现在已经变得相对擅长应对这些……我不会说他们是顶尖的工程师，但他们很擅长在复杂的系统里摸索，并搞清楚自己需要去尝试什么，而且我们拥有的系统和基础设施已经变得越来越复杂。所以，开展这项工作所需的技能组合确实会随着时间推移而发生变化。

Matt:

真是太引人入胜了。好的，我们仍然留在强化学习这个话题上，并绕回到你一开始提到的一些事情。如果我想让我的模型在计算机操作、基因编码或者任何其他领域表现得更好，我是否会花特定的时间专门针对计算机操作进行强化学习，并整理出一个数据集，然后去设计奖励机制？情况是这样运作的吗？就像你只要选定一个问题，然后专门针对它进行强化学习。

Yan:

澄清一下，我多谈一些强化学习，也是因为这是我最了解的部分，而且这也是我长期以来一直致力于推动的工作。我们之前谈到了中期训练（mid-training），其实所有这些环节都极其重要，你可以在流程的不同阶段去优化模型。正如我之前所说，你越接近模型的最终阶段，通常训练的规模就会越小。因此，你可以在那上面进行快速迭代，因为现在的迭代是以“天”为单位，而不是以“月”为单位。所以人们通常会从这种快速迭代循环开始，然后不断深入，在整个技术栈中做出更大的改变。这并不是说只有

强化学习

重要，我绝对不是这个意思。只是因为这

是人们开始做出改变的切入点，随后这些改变会渗透并深入到整个技术栈中。

事情就是这样运作的。

在开源世界中，情况也基本如此。我想你看到的后训练（post-trained）模型要比新的预训练底座模型多得多。

而且你会看到算法层面的改进要多得多，这就是为什么我们之前提到了，我是说像 GRPO、DPO、PPO，有那么多各种各样的 XPO，这正是因为人们可以在流程的最后这个阶段进行非常快速的迭代。

Matt:

那么这些模型表现出的那种“锯齿状（能力不均衡）”的特性，是因为这种“挑选这个问题、那个问题”的方法导致的吗？也就是说，它会在被选中的这些问题上表现得非常出色，但在其他问题上就没那么好；还是说，这是 AI 模型一种更本质的特征？

Yan:

确实有这方面的因素。可以肯定的是，如果你针对特定类型的问题进行更多优化，你在这个场景下的表现就会更好。但我会说，至少我的直觉是，这跟你在优化的具体问题关系不大，而更多地取决于你所优化的那一“类”问题。例如，如果你的模型非常擅长数学竞赛，那么它在编程竞赛中的表现可能也会相当不错。所以这不是领域的问题，

更多的是在于所需的技能、思考方式，以及执行这些任务所需的

横向能力（horizontal capabilities）

。我认为你通常看到的情况是，

当某个模型在某件事上表现得很差时，它实际上在任何领域、任何语言中表现都很差。

所以你必须去思考这个领域以及该领域的泛化能力，而不仅是针对单个领域的能力。

泛化的关键是横向能力，能力缺乏导致 LLM 产生幻觉

Matt:

既然谈到泛化，从数学和编程的成功到现在开始涵盖不同的领域，已经有了一个明显的演变。这就是所谓的“GDP-Eval”之类的事情，在整个经济体中，不同领域的模型表现都在接受评估。类似的问题是，这究竟是模型整体进步的结果，还是刻意为之的，比如“好，我们现在要针对经济的这一部分构建一个数据集，进行中期训练和后训练”？这种从非常特定的领域向世界其他领域泛化的进展，是如何实现的？

Yan:

这绝对是我们正在积极推动的事情。我认为大家正在意识到，我是指我们以及其他公司，我们正在迈向这样一个世界：我们希望真正做出有用的产品，提高人们的生产力，并在日常生活中为人们提供帮助。因此，我认为现在全行业都在非常积极地行动，去决定我们应该优先考虑哪些领域。既然我们知道自己拥有可以应用在不同地方的算法，那么

现在的限制因素更多在于收集正确的数据，以及让真正关心某个特定问题的人去研究这个问题

。但是，能够做这些事情的人并没有那么多。所以你真的需要划分优先级。是的，这是一种非常积极、主动的方式。

总的来说，我认为模型的表现真正取决于有多少人关心模型的最终输出、有多少人在关注这个模型

。如果他们开始更多地关注特定的垂直领域（verticals），这些垂直领域就会提升得非常快。但正如我所说，我们并没有那么多能够做这些事情的人才。

Matt:

但为了进一步拆解你刚才提到的一点，现在模型是否真的展现出了更强的泛化能力，特别是从强化学习的角度来看？也就是说，让一个模型在领域 A 或 B 上变得非常好，是否可能让它在领域 C 上也表现得更好，无论你花了多少精力去为领域 C 开发奖励机制？

Yan:

我认为泛化有不同的维度。首先是

算法上的泛化

（algorithmic generalization），也就是说，我能否将我为领域 A 开发的算法或黑盒，直接应用到领域 B 上？至少在开源世界中，人们似乎确实能够做到这一点。他们把 GRPO 拿过去，应用在很多不同的地方，它就能直接奏效。所以这种算法层面的泛化看起来相对较好，这也是为什么我们能看到如此多进展的原因，否则取得进展会非常困难。

其次是

基于特定数据集训练出的模型的泛化能力

。这就是我之前提到的，至少在我的心智模型中，

泛化是基于“能力”而非“领域”发生的。如果所需的核心能力是相同的，你就会看到跨领域的泛化

。再举个例子，就像不同的语言或编程。即使在 C++ 上的训练非常少，你也可以通过优化让它成为一个很好的 C++ 模型。部分原因是因为预训练模型已经见过了所有的 C++ 内容，所以它已经在某种程度上理解了该语言的基础知识。所以，这种类型的泛化是绝对存在的。

然而，

我认为更难的泛化，是当模型缺乏这些“横向能力”的时候。

我给你举个具体的例子。如果我的模型在竞赛（比如数学竞赛、编程竞赛）中表现得非常聪明、准确率极高，我经常举这个例子，因为它很有启发性。从人类的视角来看，擅长这些的人通常“就是聪明”；或者至少有人会认为，既然他们很聪明，他们自然也能做其他事情。但对模型来说，这并不是真的，那种跨能力的泛化并不成立。因为在许多需要人类在专家领域工作的场景中，真实世界是非常混乱的，而那些编程和数学竞赛的题目则是定义极其清晰（well-specified）的。

在现实中，你需要具备理解“定义模糊的任务（underspecified tasks）”的能力，懂得如何应对混乱的现实世界，并弄清楚解答问题究竟需要哪些资源

。比如你看数学竞赛，所有的信息通常都包含在提示词里了，可能只有 5 行或 15 行字，里面就是你回答这个问题所需的全部信息。但在现实世界中，如果我是一名顾问或者在金融行业工作，我需要上网去搜索、查找并提取不同的信息来理解背景。在进行任何逻辑推理之前，我必须先做到这一点才能开展推理。

而这种横向能力，才是决定能否泛化的关键。如果你拥有了这种横向能力，你就能泛化；但在很多情况下，我们现在并没有这种横向能力

。

这就是为什么大语言模型在各个领域都会产生幻觉的原因。如果一个模型很不擅长承认自己“不知道”，这通常会在每一个领域都发生。你不会看到模型在某一个领域对自己的知识库有着完美的校准（calibrated），而在另一个领域却胡说八道。

SFT 会迫使模型产生幻觉，强化学习能抑制幻觉现象

Matt:

顺便插一句，解决幻觉问题是否也是一个强化学习的任务？即当模型遇到不知道的事情时，你通过奖励它说“我不知道”来规范其行为。

Yan:

约翰·舒尔曼（John Schulman）大约在一两年前做过一个非常精彩的演讲。他在演讲中提到，如果你做“行为克隆”（也就是我们之前谈到的 SFT），你基本上是在奖励并优化“幻觉”。因为可能会发生这种情况：如果你的模型原本不知道某件事，但现在人类提供的地面标准答案（ground truth）却要求模型说出这件事。

我举个非常具体的例子，如果模型根本不知道某篇论文的存在，而在人类给出的标准答案中却写着：“这是我获取信息的地方”，并引用了那篇论文。那么你实际上在优化和训练模型去做的，就是去引用一个原本对它来说不存在的东西，因为它根本不知道那篇论文的存在。所以约翰在那次精彩的演讲中指出，

SFT 会迫使模型产生幻觉

。

而在强化学习中，正如我所说，你首先需要从模型本身进行采样。模型采样出一个它不知道却又是正确答案的概率是极低的，几乎不可能。所以你永远不会奖励这种行为。你只会采样到它不知道且回答错误的情况，然后你就会扼杀掉这种胡说八道的行为。所以关于幻觉，现在大家的直觉是，它可能是由 SFT 或是后训练流程中的某些缺陷引起的；但

如果你有一个出色的强化学习流程，幻觉发生的频率不应该太高。

“太听话”的模型解读不出人类的潜在意图

Matt:

再次回到泛化的话题，是否存在这样的例子：模型在某一个领域变好，反而导致它在其他领域的表现变差了一点？就像你刚才说的，有些人很擅长数学，有些人很擅长英语，而在现实中，这两类人通常不是同一拨人。

Yan:

通常不会这样。实际发生的情况是，你会根据你选择优化的领域做出权衡和决定。如果你重点优化一个领域，你投入到另一个领域的优化精力自然就会减少。所以，这并不一定是优化一件事会让另一件事变差，而仅仅是因为结果上你无法对其他事情投入同等的优化，因为你受到了算力、数据的限制，并且在工作层面上也面临着人力瓶颈。但是，

确实会发生某种“负泛化（negative generalization），或者更准确地说是

负迁移（negative transfer）

，这更多发生在模型的横向能力层面

。我给你举个非常具体的例子：显式指令遵循（explicit instruction following）与隐式指令遵循（implicit instruction following）。比如我们经常听到关于 OpenAI 模型的评价，如果你极其精确地告诉它们你想要什么，它们往往表现得非常出色。但硬币的另一面是，有时我们也听到，如果你没有说得那么具体，它们的表现就会打折扣。例如，

如果我想修改某个文件，但在输入文件名时打错了字。一个极其擅长“显式指令遵循”的模型会不折不扣地去修改那个包含错字、实际上可能并不存在的文件。

而如果是人类，可能一眼就会意识到你打错字了。因此在某些情况下，

这种对显式指令的极致遵循，反而会与理解隐式指令的能力发生冲突。所以确实会出现不同的横向能力相互冲突的情况。

Matt:

或许我们可以用这个问题来结束关于强化学习的讨论。随着我们从精通编程、精通数学逐步走向经济体的其他领域，你认为其他这些领域的难题是可攻克的（tractable）吗？你认为我们最终在这些领域能达到同样高的性能水平吗？

Yan:

可以。但我会说，是的，我们可以。我不认为这些领域有什么极其特殊、深奥的地方，以至于我们无法进行优化或无法取得与其他领域相同的成果。但我说“但是”，至少有两个原因。首先，开发这些模型的大多数人本身就非常擅长编程，而且他们极度关心编程，因为那是他们日常工作的主要驱动力。没有什么比让用户同时作为模型的训练者更棒的了，因为这样他们能切身理解问题所在。相反，对于我来说，

如果我对法律领域一无所知，我就很难真正搞懂我们应该在模型的法律垂直领域去修改和优化什么。

这是第一点。

另一点是你经常会听到的、我之前也简要提过的，就是所谓的

可验证奖励（verifiable rewards）

。在某些领域，判断某个结果是对是错要容易得多。例如在网络安全领域，

你之前提到过我们模型的网络安全能力提升了很多，这是因为在网络安全中，判断你是否正确是极其简单的：比如你发现的网络安全漏洞到底是不是一个真实的漏洞，这非常容易测试。

因此，确实有一些领域更容易应用强化学习。但我想说，

从模型的潜在能力来看，没有任何因素限制它在法律、医疗或其他领域达到同样高的水平

。简而言之，就是我们对这些领域的了解还不够深入，而且确实有一些领域在强化学习中更容易进行优化。

模型评估愈发成为瓶颈，

让模型成为裁判，产生能力飞轮

Matt:

太好了。我们来聊聊评估（evals）。这是一个极其重要的话题。也许我们可以先从这里开始：为什么评估一个模型最初会这么困难？

Yan:

随着模型变得越来越好，评估也变得越来越困难。这是因为我们要求模型执行的任务正变得越来越通用、越来越开放。

比如现在我可能会说“帮我建一个能实现 X 功能的网站”，而在过去，我可能只会问“嘿，在这个具体的代码实现里有没有特定的 Bug？”。判断有没有 Bug 要容易得多，因为我可以提取信息，让一个基准人类指出所有的 Bug，然后自动应用这一标准。而建网站的任务很难知道什么是“最优解”，因为有很多种好答案，也有很多种搭建某个网站的好方法。模型的这种开放性确实让评估变得更难了。

另外还有一个问题，

那就是模型在某些特定维度上正变得比大多数人类还要好，所以我们能够真正去评估模型在这些维度上表现的人类变得越来越少

，这绝对是一个限制因素。老实说，另一个原因是文化层面的。大多数人都想提升模型，他们认为最好的方法就是去训练模型，但现实中，发现问题并确保我们能量化这些改进其实同样重要，甚至更重要。然而，这种文化认知上的差距一直存在。这在学术界尤其明显，直到大约两年前，评估总是固定的，基准测试总是固定的，甚至连数据集在可能四年前也是一成不变的。后来大家的观念发生了转变，意识到“数据实际上至关重要”，现在有很多力量在研究数据。但我认为

对评估的重视仍然不够

。大家都知道它重要，但人们并没有真正完全理解投入到评估中能带来多大的影响。所以实际上，我在 OpenAI 的第一个项目，我一进来就说“我想做数据和评估”，因为我知道这是没人愿意碰的领域，正因如此，我知道做这个会带来超级巨大的影响力。是的，现在风向正在转变，但速度还不够快。

Matt:

那么“模型作为裁判（Model as a Judge）”以及“AI 评估 AI”这一领域的进展速度，是否也同样迅速？这是一个独立的研发分支，还是在本质上基于相同的理念或技术？

Yan:

本质上完全是相同的方法。而且我们在评估中所做的绝大多数事情，特别是现在我们有了强化学习，几乎可以原封不动地直接应用到训练过程中。其实这也是评估如此复杂的另一个原因：

你每次建立一个评估体系，实际上就是在建立一种构建训练数据集的方法。

接下来你就会针对那个训练数据集进行优化。即便它不是那个评估本身，也是同一类型的数据，然后由于我之前提到的那种核心能力的泛化，模型会表现得非常好。模型会在另一个数据集上学到这些，进而在当前的评估中拿到高分，导致那个评估指标极快地过时。这也是评估所面临的一个头疼问题。

不过回到你的问题，

模型作为裁判确实非常重要，我认为它可能是最关键的事情之一。

因为

随着我们拥有更好的模型，就会形成一个自我强化的闭环，产生一种能力飞轮：更优秀的模型会成为其他模型更好的老师。

这在训练中极其重要，同样你也可以把它应用到评估上。所以我的团队中有很多骨干正在做这个，我认为构建“模型作为裁判”这种框架是绝对关键的。

未被攻克的难题：持

续学习让 Yan Dubois

感到极其兴奋

Matt:

好的，太棒了。那么随着我们今天的对话接近尾声，我想把视角拉远一点，听听你对未来的走向有什么看法。显而易见，对几年后的 AI 发展做预测是极度困难的，但如果我们把时间拉近到未来的 12 到 18 个月，或者 24 个月。在你的直觉中，技术是会继续以渐进的方式进步，还是我们正走向某种让人感觉是断裂式的、非连续性的飞跃？

Yan:

就进步本身而言，正如我之前所说，我认为它始终是连续渐进的。不过，确实会有让人产生“非连续性”感觉的时候。三四个月前在编程领域就发生了这种变化，我认为这种感觉现在正在向所有其他领域扩散。

大多数人目前对模型能力以及实用性的感受，还没有达到像现在编程和软件工程领域的那种颠覆感。

所以我认为这绝对会渗透到许多其他垂直领域。而如果从能力跨越的角度来看，比如在我们已经在关注的一些垂直方向上，我认为发展会更加连续，不会出现巨大的断裂。它们大多数只是局部看似不连续，但当你拉长视角、放大宏观层面时，曲线其实一直非常平滑。虽然不绝对，但大多数时候确实如此，我也完全无法预测下一次巨大的非连续性飞跃会何时出现。

Matt:

你对“AI 中的加速循环（accelerating loops）”这一普遍概念有什么看法？无论是通过持续学习让模型保持最新状态并学得更快，还是更宏大的“AI 以日益自动化的方式构建 AI”的概念，这到底已经是事实，还是仍属于科幻？有哪些是让你感到兴奋的？

Yan:

我对持续学习（continual learning）感到极其兴奋。我认为我们还没有完全攻克它。我的意思是，我们确实有一些像记忆库（Codex memories）之类的东西，这很有帮助，但绝对还不是终局形态。我有一个朋友总跟我提到另一种我们应该关注的图表：横轴是时间，纵轴是模型提供给用户的效用（或者基本可以说是模型的实用性）。目前，在第 0 天，如果你把模型直接放到一家公司里，可以毫不夸张地说，它们往往比绝大多数新员工更有用，所以它们在初始时刻的起点很高。但随着时间推移，它们的表现基本保持不变，因为它们并不会真正学习公司的内部知识，也不会随着时间推移在执行现有工作时变得越来越高效。然而人类学得非常快，而真正核心的指标是这些曲线的积分，也就是曲线下的面积。这就是为什么在很多情况下，人类依然更好用。因此，我

们需要通过持续学习来让这条曲线随着时间推移保持单调递增，基本上就是让模型在某个特定环境中工作得越久就变得越好用。

所以我对此非常兴奋，其实我也很惊讶我们居然还没走到这一步。

三年前 ChatGPT 问世的时候，我记得当时我和朋友正在做一家初创公司。我们当时考虑研究持续学习、个性化以及通用的长短期记忆。我们当时想：“啊，OpenAI 肯定会在接下来的六个月里把这个做出来。毕竟他们拥有所有的数据，有那么多的用户，而且模型能从用户那里学得超级快，他们肯定能解决。”结果三年过去了，我认为我们依然还没做到。

Matt:

用通俗的话快速解释一下，最根本的难点在哪里？

Yan:

问得好。跟你说实话，我其实不太清楚。我确实不太明白为什么我们花了这么长时间还没搞懂它。我认为这是一个只要我们真正投入足够的资源就一定能解决的领域。当然，特别是当传统谈到公司内部的这种记忆库时，会涉及到巨大的合规权限问题，以及关于隐私的许多疑问，比如在不同的用户之间（对不起，是用户之间，而不是模型之间）什么可以共享、什么不能共享。但就算是对单个用户，甚至纯粹针对单用户，我们也没能真正做到。我不太清楚其中的原因，至少在我能谈论的高层面上，我不知道为什么。

底座模型越来越强大，Agent Harness 会一直处于变动之中

Matt:

是的，你提到的这一点，我认为对于 AI 开发者、投资者以及初创公司来说都非常有趣。那就是模型在企业内部正变得越来越聪明，特别是模型本身能做的事，与许多人围绕模型构建的外围体系之间存在着博弈与张力。你知道，一两年前大家都在做 RAG（检索增强生成），而如今所有人都在卷Agent Harness。很多人都在好奇，模型最终是否会把这些

Harness

给“吞噬掉”，这些 Harness 到底是不是一个阶段性的过渡产物？从你的视角来看，你觉得未来会发生什么？

Yan:

是之，我认为

Harness

在现阶段确实能显著提升模型的能力。但是，鉴于我们看到模型核心能力正以极快的速度演进，我个人不会过度把宝押在

Harness

上，除非这个

Harness

是纯粹为了你当下试图实现的某个非常具体的业务目标而服务的。比如某些专注于特定垂直领域的公司，他们希望将模型的可靠性从 80% 提升到 85%，

Harness

确实能帮他们做到这一点，我认为这非常重要。但他们在做这件事的同时必须清楚地意识到，未来他们肯定需要重新调整这个

Harness

，我认为这完全没问题。

如果你试图做一套通用的、想在未来长盛不衰的

Harness

，我认为是行不通的,针对特定领域的特定

Harness

只能算是一种短期内的权宜之计。我想在

Harness

层面能做的事情总是非常多的，如果有什么建议的话，我认为如果大家脑海中有一个具体的问题，每个人都应该在这上面多做尝试，因为如果没有一个好的 Harness 配合，我们其实浪费了模型很大的潜能。可以这么说，如果我们现在就把现有的模型完全冻结不再更新，然后大家开足马力拼命去优化

Harness

，或许再加上多花时间用优秀的

Harness

去训练它，我认为大家在每一个单点领域里都会真正感受到 AGI 的降临，或者说已经能够感受到了。但现实是模型并没有被冻结，我们还会继续训练越来越强大的底座模型。因此，

我认为我们现在还远远无法断定最终的

Harness

会是什么形态，它会一直处于变动之中
