# 微软用Go语言卷智能体了，谷歌也已入局，只剩OpenAI和Anthropic了

> 原文链接：https://mp.weixin.qq.com/s/NgOAAKCJ4s_MLJ1YLZmxcQ
> 公众号：InfoQ

作者丨 Paul Sawers

译者丨明知山

策划丨 Tina

Go 语言已成为云基础设施领域的通用标准语言，从容器编排、CI/CD 流水线，到工程师日常离不开的各类命令行工具，无处不在。Kubernetes、Docker 和 Terraform 都是用 Go 编写的，时至今日，它仍然是团队构建后端服务的首选语言。

与此同时，AI 智能体已成为几乎所有主流软件厂商的焦点，各家都在竞相为开发者提供原生构建智能体的框架。正因如此，微软现在正将自家智能代理框架引入 Go 语言生态，让云原生开发者能够用日常开发各类业务的 Go 语言来构建 AI 智能体。

最近开始公开预览的微软 Agent Framework for Go 本质上为 Go 语言开发者提供了许多与 Python 和 .NET 开发者相同的构建模块：支持来自 Microsoft Foundry、Azure OpenAI、Anthropic 和 Gemini 的模型；支持工具调用和 MCP，可实现智能体连接外部系统；同时能够调度多个智能体协同完成同一任务。

微软高级软件工程师 Quim Muntal 在发布官方博文中提到，这个智能体框架面向的是那些构建功能更强大、运行周期更长的 AI 系统开发者。

Muntal 写道：“Microsoft Agent Framework 专为那些正从单一提示词调用转向生产级智能体系统的开发者而设计。这些智能体可调用工具、保持上下文信息、与其他智能体协调、流式传输结果，并且能够作为真实业务应用的一部分接受监控和治理。”

这正是 Go 语言已经占据的领域：后台服务、命令行工具、后台任务程序和云原生应用，长久以来 Go 都是这类场景的首选开发语言。本次推出的 Go SDK 将这些能力直接交到在该领域构建的开发者手中。

1

微软 Agent Framework 适配 Go 语言的历程

简单回顾一下：微软于 2025 年 10 月推出 Agent Framework，作为一个用于开发 AI 智能体与多智能体系统的开源工具包，将此前的两个微软项目 AutoGen 和 Semantic Kernel 统一为一个受支持的平台。

它在今年 4 月份正式全面可用，在 6 月份的 Build 开发者大会上，微软又增加了一系列新功能和工具：用于生产环境（如上下文管理和人工审批步骤）的“智能体 harness”；通过 Microsoft Foundry 提供的托管智能体；一种更快的工具调用方法 CodeAct；以及支持通过新的交接模式将多个智能体串联在一起。

在此之前，Agent Framework 仅支持 .NET 与 Python 两种语言。社区中有不少开发者都呼吁官方适配 Go 和 Rust。早在 2025 年 10 月，微软产品经理就曾证实团队正在考虑推出该框架的 Go 版本，但当时表示距离上线“至少还需要数月时间”。

目前框架尚处于起步阶段，但开发者社区已经出现不少讨论，分析此次发布具备哪些价值、又存在哪些局限。AI 工程师 Pratik Dhanave 在领英发文称，这对以 Go 语言为主要开发语言的开发者而言是一个里程碑，并特别提到该软件开发工具包基于图结构的工作流编排能力，支持条件分支路由、子工作流、断点续存、人工介入审核等开发范式。

他写道：“这是生产级的编排工具，不只是对对话循环的简单封装。”

Dhanave 同时提到，微软官方文档也写明了这款 Go 开发工具包目前尚不具备的功能：任务交接编排、CodeAct 等特性目前仅支持.NET，尚未移植到 Go。他接着表示：“在公开预览版中能做到如此坦诚地说明短板，这点我十分认可。”

2

谷歌、OpenAI 和 Anthropic 在 Go 语言方面的布局

谷歌方面的发展路径与之相近，只是推进节奏有所不同。谷歌于 2025 年 4 月份推出自研智能体开发工具包（ADK），初期仅支持 Python，随后于同年 11 月份新增 Go 语言支持，今年 3 月份正式发布功能完整的 1.0 稳定版本。

Go 语言出自谷歌，2009 年从公司开源项目库中诞生，这一举措塑造了后续整个云原生技术生态。此后，Kubernetes 与 Docker 两大奠定现代基础设施构建、部署标准的项目均采用 Go 语言开发，这也让 Go 成为平台工程师的首选开发语言。它的影响力早已跳出谷歌，延伸到了亚马逊云科技、Cloudflare、微软以及无数其他科技巨头。

鉴于 Go 语言如此强大的吸引力，开发者迫切需要智能体工具提供原生 Go 语言支持也就不足为奇了。此前，想要开发智能体服务、在现有微服务中嵌入 AI 能力，或是编写具备自主决策能力的基础设施工具的 Go 开发者，一直没有官方配套 SDK 可用，只能另寻变通方案：直接调用底层 HTTP 接口、在 Go 服务之外额外运行独立的 Python 或 Node.js 进程，或是依赖非官方、由社区维护的第三方库 —— 这类库质量参差不齐，技术支持也没有保障。

由此可见，微软此举显然是在服务规模庞大、根基稳固的 Go 语言开发者群体。但 AI 领域的其他头部厂商尚未跟进。截至目前，尽管社区开发者多次呼吁适配，Claude 智能体开发工具包和 OpenAI 智能体 SDK 均未官方支持 Go 语言。

并非只有微软一家为 Go 语言推出 AI 智能体开发框架，但微软此举意味着两大头部云厂商如今都已为这款业界应用最为广泛的基础设施编程语言提供官方原生智能体开发套件。而基础大模型领域剩余两大头部厂商 —— Anthropic 和 OpenAI 在这一方面已然落后。
