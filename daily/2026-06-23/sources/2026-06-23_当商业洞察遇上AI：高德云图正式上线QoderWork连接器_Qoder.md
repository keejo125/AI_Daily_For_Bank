# 当商业洞察遇上AI：高德云图正式上线QoderWork连接器

一个做了十年餐饮的人，决定开第三家店。

他比任何人都了解自己的菜品、自己的顾客、自己的供应链。但对于"第三家店开在哪"这个问题，他能依赖的信息其实非常有限——朋友的建议、中介的推荐、自己在目标商圈蹲了几个周末的观察。

至于大数据？他知道那东西有用。但怎么用、找谁用、花多少钱用，他从来没搞明白过。

这不是一个人的困境。在中国数以百万计的中小商户和连锁品牌里，能用上商业地理数据洞察的，始终是少数。不是数据不存在，是门槛太高了。你需要懂技术、有预算、有团队，才能把数据变成决策依据。

降低这个门槛

基于空间智能，服务 AI Native 场景

高德云图的核心能力是空间智能——

用 AI 理解物理世界中各种真实要素之间的关系。

围绕这个能力，我们为 AI Native 的应用场景提供了多个方向的洞察服务：交通出行领域的态势分析，商业洞察领域的选址评估与经营诊断，文旅领域的城市消费与景区洞察。这些方向共享同一套空间智能底座，面向不同的行业场景输出差异化的分析能力。

为了让这些能力真正被 AI 用起来，我们构建了三层开放体系。

行业 MCP 服务

，是数据能力的标准化开放层。把商业、交通、文旅等行业的核心数据能力封装成标准 MCP 接口，让任何支持 MCP 协议的 AI 工具都能直接调用。无论是商业选址、交通研判还是文旅分析，都可以通过统一的接口接入。

专家 Skill 技能包

，是场景化的能力封装。针对不同行业的典型业务场景——选址评估、经营诊断、通勤分析、景区客流研判——把多个数据能力和分析逻辑打包成可直接使用的技能，降低从"有数据"到"会用数据"的落地成本。

企业级 Agent 智能体

，是面向终端用户的直接入口。用户用自然语言提出业务问题， Agent 自主完成从数据查询到结论输出的全流程。不懂技术的人也能直接获得专业级的分析结果。

三层能力互相配合——底层数据通过 MCP 开放，场景能力通过 Skill 沉淀，终端应用通过 Agent 触达。这套体系覆盖交通、商业、文旅等多个方向，目标是让空间智能洞察不再是少数人的特权。

为什么是 QoderWork

QoderWork 是阿里推出的桌面 AI Agent 。和传统的 AI 对话工具不同，它能理解业务意图，自主规划分析步骤，调用连接好的工具，把结果整理成可以直接用于决策的结论。

对使用者来说，整个体验就是：说一句话，等结果。不需要写代码，不需要懂 API ，不需要在多个系统之间来回切换。

当商业洞察 MCP 接进 QoderWork 的连接器，意味着我们的需求分析、 POI 搜索等商业数据能力，通过 MCP 这层开放接口，被 QoderWork 这个 AI Agent 平台调用了。一个品牌拓展负责人、一个区域运营经理、一个小店老板，都能在 QoderWork 的对话框里直接使用这些分析能力。

MCP 是第一步。随着 Skill 和 Agent 能力的后续接入， QoderWork 上的商业洞察体验还会更完整——不只是能查数据，还会有封装好的场景化技能和更智能的端到端分析。

这是 MCP + Agent 组合的真正意义：数据能力从"开发者专用"变成了"所有人可用"。

不列功能清单，直接看场景。

选址评估。

"帮我查一下上海黄浦区、静安区、徐汇区上个月工作日的客流规模和消费水平，做个对比。" QoderWork 自动调用云图能力，返回三个区的多维度对比数据。选址粗筛的第一步，过去可能需要一个数据分析师半天，现在一句话。

商圈洞察。

"搜索杭州天目里，分析这个区域上周末的需求特征。" 先定位地点，再拉取区域相关的多维洞察。品牌和目标商圈的匹配度，一目了然。

时段分析。

"成都春熙路商圈晚上 6 到 9 点的服务需求和工作日中午整体有什么差异？" 云图支持小时级精度，不同时段的需求结构的差异直接可见。门店排班、促销时段，不用凭经验猜了。

经营复盘。

"对比北京三里屯商圈今年 4 月和 5 月的客流变化，重点看下午时段。" 月度趋势加上时间切面，快速定位客流变化的时间窗口。

这些场景有一个共同点：过去需要专业团队、专门工具、专门时间才能完成的工作，现在任何人在对话框里说一句就能拿到结果。
