# Qoder 负责人丁宇在2026世界人工智能大会主旨演讲：打造安全可信的智能体

> 原文链接：https://mp.weixin.qq.com/s/HGDG8DB68PTlz4puWmWriw
> 公众号：Qoder

今天想和大家聊一件正在发生的事——智能体正在重新定义数字世界的生产力和生产关系。

过去我们谈 AI，它活在对话框里。而今天，智能体正在跳出对话框，找到新的环境和土壤：它进入浏览器、App、电脑、手机，进入组织的真实工作流，从“陪你聊天”变成“替你干活”，创造出新的形态和新的价值。

这背后，

Coding Agent 正在成为数字世界的基础设施。

因为 CLI 本就是大模型最自然的运行环境，Coding Agent 也因此成长为通用智能体，支撑起整个 Qoder 产品家族。它具备四种关键能力：编排，一个任务自动拆分成多个 Agent 并行、完成后自动合并，甚至能在手机上远程控制；集成，多种模型随时切换，通过 Headless、ACP、SDK 三种形态被灵活集成；扩展，用 Skills 沉淀最佳实践、Hooks 审查行为、Plugin 封装能力；以及自治，Agent 自主判断风险，不再逐条打扰你，只在关键时刻请求确认。

要想做好基础设施，需要一套安全可信的架构。在企业环境中引入 AI Agent，本质上是在向一个自动化主体开放代码、数据与生产系统的操作权限。权限的开放程度，直接决定了风险敞口的大小。 因此，Qoder 在架构设计上遵循一个基本原则：能力必须运行在可控、可审、可回溯的边界之内。

整个架构分为两层。上层是对客安全，覆盖 AI 工作的全生命周期，对应企业最关心的四个控制点。

接入层，解决"谁在用、用什么"的问题，提供身份安全、多租户隔离与模型自定义能力，确保租户间数据与算力严格隔离。

运行层，管控 Agent 的实际行为，涵盖生态扩展安全、代码安全与 Agent 执行安全，保证每一次工具调用和代码操作都处于策略约束之下。

数据层，落实数据合规要求，通过数据隐私保护、敏感信息保护与敏感行为识别，实现数据不出域、敏感操作可拦截。

审计层，满足企业的合规与追责需求，提供全链路追踪、不可变审计日志，并支持与企业现有 SIEM 系统集成，做到全程留痕、事后可查。

下层是安全工程底座，这是支撑上层能力的内部工程体系，贯穿产品自身的研发运维全流程。

设计阶段坚持安全架构先行、默认安全与多层防御；开发阶段实施模型安全、代码安全与内容安全的多重校验；运行阶段落地账号风控、多租安全与存储加密；监控阶段建立应急响应、漏洞管理与安全运营机制。

这两层的关系可以概括为：对客安全是我们向企业交付的安全承诺，安全工程底座是我们兑现这一承诺的工程能力。 从纵深防御到可信合规，

Qoder 的目标是让企业在享受 AI 生产力的同时，始终掌握对边界的控制权。

更进一步，我们把这套能力做成了 Qoder CLI S

DK——给软件装上大脑。几行代码就能集成 Agent，让智能进化成为软件的原生能力。当软件有了大脑，它可以按需改写自己，实现生长和自进化；一个 SDK 两种形态，任务在本地和云端之间无缝迁移。这不只改变了人使用软件的方式，也改变了构建软件的方式。

面向开发者，我们提供 Qoder Desktop，把个人的开发经验变成组织共享的工程知识引擎。你可以用 Prompt 定义专家角色、用 Skill 完善协同、用 MCP 连接工具，组建适配自己团队流程的 Agent 专家团。当智能体生成代码的数量是手写的一百倍，我们需要的是一个 Agent 优先的研发工作台。

规模越大，安全越重要。Qoder 安全能力全面升级，把代码安全从“静态扫描”推向“主动式安全”：基于自研大模型，生成即审查，L1 实时拦截、L2 语义级增量审查、L3 跨文件深度追踪，构建发现、反馈、修复的完整闭环。漏洞检出率提升到 90%，修复时长从数天压缩到数小时，成员零配置就能用。
