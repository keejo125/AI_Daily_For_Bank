# Qoder CLI /goal上线，你定目标，Qoder达成

> 原文链接：https://mp.weixin.qq.com/s/IQon-sCk7Nchgq3ODVmggQ
> 公众号：Qoder

1 images

和 AI 协作到现在，很多时候我们依然被困在"发 prompt → 等结果 → review → 再发 prompt"的循环里。一个简单重构，被切成几十轮对话。

今天，Qoder CLI 上线 /goal，你只要定义目标和完成标准，剩下的拆解、写码、跑测试、看结果、修 bug、再跑测试，全部在循环里自动推进，直到验证条件满足为止。

普通 prompt 是一次对话：你说一句，AI 答一句，/goal 是一个长程目标：

​

1. 走开也能进展。 进入qodercli后，敲下 /goal set "把支付模块迁移到新 SDK 并补齐单测覆盖率到 80%"，然后你可以去开会、去吃饭、去睡觉。回来时打开终端，看到的是已经跑通测试、提好 PR 的状态。

2. 跨会话持久化。 目标挂在 thread 上，不依赖你这一次终端有没有关。第二天接着跑，上下文、已完成步骤、剩余任务都在。再也不用每天早上花 20 分钟"对一下昨天 AI 改到哪了"。

3. 自带验证闭环。 每一轮迭代结束，/goal 会按你设定的成功条件自检：测试有没有过、覆盖率达没达到、lint 有没有报错。证据不达标就继续，达标了才停。

4. 为长任务而生。 修一个跨多个文件的 bug、把一个老模块重构掉、给一个仓库补齐文档——这些以前要拆成无数 prompt 的活儿，现在一句 /goal 就能托管。

升级到最新版 Qoder CLI，立即体验！
