# 投了OpenAI 130亿的微软，被自家工程师用Claude Code烧爆了账本

> 原文链接：https://mp.weixin.qq.com/s/_nGuQ45PFecEDU6eYPSiGg
> 公众号：新智元

新智元报道

【新智元导读】

Claude Code按toke计费，微软先崩了！工程师沉迷Claude Code，6月底全员强制断供。

微软工程师们，天塌了！

昨天， 微软工程师们还在用Anthropic的Claude Code狂飙代码、改代码库、跑代理，今天就被一纸通知：6月底前必须全部下线，强制转向自家GitHub Copilot CLI！

这不是小打小闹，是微软多个部门集体断供Claude Code。

更狠的数据锤来了

：连微软这种近乎无限云资源的巨头，都觉得Claude Code的账单太难看了。

同一时间，Uber CTO Praveen Neppalli Naga内部备忘录更直接捅破天花板：

2026年全年AI预算，才过了4个月就彻底烧光！

这不是AI在帮忙，这是AI在烧钱啊！

AI补贴时代正式终结,真实的「计算账单」开始压垮企业预算。

当微软和Uber这种不差钱的巨头都开始喊「用不起」时，意味着AI正在从效率工具变成吞金黑洞。

这不再是技术实验，而是关乎所有企业现金流的生存危机。

Token费用，微软顶不住了

外媒爆料显示，微软计划在6月底前停掉所有员工的Claude Code订阅。

具体范围已经划定：负责Windows、Microsoft 365、Outlook、Teams和Surface 的「体验+设备」部门。

他们须在6月底前停用Claude Code，并在未来数周内把工作流迁移到自家的GitHub Copilot CLI。

近10万名微软工程师，将被要求全员转投。

理由极其直白：账单太贵。

这次断供被内部定义为一次「紧急刹车」，而非有序的工具迁移。

去年，纳德拉披露「30%代码由AI生成」，但token计费框架显露出残酷的一面。

传统软件开发里，工程师写得越多，公司赚得越多；在AI智能体时代，AI 写得越多，付给外部供应商的账单涨得越快。

每一个Token的生成，都在实时消耗微软的现金流。

这就像雇了一个效率惊人的保安，却按他出勤的每一分钟精确计费——当他干活比正规军还勤快时，雇主的财务平衡反而被打破了。

据媒体还原的内部沟通，这次断供由财务部门主导：工程线一致反馈生产力提升巨大，管理层却选择了强制下线。

这已经超越了产品偏好之争。

账本逻辑，最终战胜了工程师的技术品位。

最讽刺的是，微软给OpenAI砸下130亿美元，还亲手在Azure上为竞对Anthropic搭建了大部分算力底座。

但当自家工程师大规模调用Claude Code时，财务流程透明而冷酷：微软需要按Token向Anthropic付费——等于在给自己最直接的竞品输血。

看完账单后，这家市值3.5万亿美元的巨头，决定不再当这个「冤大头」。

对微软而言，GitHub Copilot属于「内部核算成本」，资源在Azure内部流动，边际成本极低；Claude Code则是一笔实打实的「外部账单」。即便两者产出完全相同，财务性质也天差地别。

当然，微软给出的官方辞令是「战略对齐」。

辞令是体面的，账单是藏不住的。

这是AI工具市场第一次大规模的财务纠偏。

四个月，Uber烧光全年预算

微软的遭遇并非孤例。

Uber更夸张：在Token计费时代，最积极拥抱技术的客户，可能最先撞上财务危机。

Uber首席技术官 Praveen Neppalli Naga 的内部备忘录，令人心惊：

95%的工程师月活使用AI工具，

84%已进入「智能体编码（Agentic Coding）」模式，

70%的线上提交代码源自 AI生成。

但代价是灾难性的。

原本留给2026年全年的 AI 专项预算，在前四个月就消耗殆尽。

财务团队基于「固定席位」「低频调用」建立的预测模型，在面对5000名工程师并行开启的智能体工作流时，彻底失效！

在智能体模式下，重度用户的月度成本最高可达2000美元/人，而这种支出与生产力产出之间，尚未建立起稳健的对等关系。

Uber的案例释放了一个系统性信号：

用得越深，烧得越凶。

当企业财务团队还没学会管理这种「效用制」定价时，AI 的效率神话往往伴随着预算黑洞。

范式地震：从「包月自助」到「按克计费」

过去三十年，我们习惯了软件业的一种慷慨：付了月费，软件就像呼吸一样廉价。这是 SaaS（软件即服务）的底色。

但AI正在暴力拆毁这套逻辑。

Anthropic、OpenAI和微软旗下的GitHub集体转身——GitHub已经取消「一口价」固定计划，全面转向按使用量计费。

过去半年，全球AI软件的有效价格上涨了20%到37%。

人类正式从「订阅经济」跨入了「效用经济（Utility Economy）」。

以前，用AI像吃自助餐，付了门票，扶墙进去，能吃多少全看胃口。

现在，AI变成了市场里按克计价的顶级和牛，明码标价。

每一行自动生成的代码，每一个被调用的Agent，都在计费器上留下一串血色的数字。

智能不再是一件静止的工具，而是一种高能耗的「代谢过程」——每一口呼吸，都要付费。

账总得有人来算

那么，这场赛跑会跑向哪里？逻辑上只有两条路，而它们殊途同归：

其一，企业缩减AI用量以符合预算——但这会拖慢AI实验室的营收增速，进而撑不起它们IPO前的高估值。

其二，AI实验室主动降价、自己吞下亏损——但这会在最糟糕的时点，进一步恶化它们本就脆弱的单位经济。

无论走哪条，终点都一样：账算不通了，总得有人为这笔亏损买单。当下AI的估值，是建立在「收入会持续爆发」的假设之上的。

一旦企业因为太贵而收手，或AI公司为留客而亏本甩卖，悬在整个行业头顶的，就是那两个字——减值（Writedown）。

对每一个企业主来说，指挥棒正在易主。

过去，AI的推广由CTO和开发者驱动，追求的是「快」和「酷」；从现在起，CFO可能成为AI团队真正的最高指挥官。
