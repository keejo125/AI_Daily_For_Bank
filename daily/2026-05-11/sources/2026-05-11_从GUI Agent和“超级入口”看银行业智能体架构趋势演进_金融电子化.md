# 从GUI Agent和“超级入口”看银行业智能体架构趋势演进

> 原文链接：https://mp.weixin.qq.com/s/KpPH1-k7XBKuaZRh1o0v-A
> 公众号：金融电子化

文/中国邮政储蓄银行金融科技部   黎育龙   高运凯   陈鹏

国务院《关于深入实施“人工智能+”行动的意见》提出，在金融等领域推动新一代智能终端、智能体等广泛应用；到2027年，应用普及率超70%。2025年12月，“豆包手机”发售，引发业界关注。“豆包手机”的核心技术组件是系统级的GUI Agent（图形用户界面智能体）。这一系统级的智能体，基于豆包大模型，通过感知屏幕、规划步骤、模拟点击等方式，实现跨应用的自动化执行。例如，用户发出“帮我全网比价并下单”的指令，GUI Agent能自动串联多个购物App，完成选品、比价、下单等一系列操作。“豆包手机”赋予智能体操作系统级别的高权限，使其能够模拟人类“读屏”并进行“点击”操作。这是端侧AI交互的重要突破，加速了交互范式从GUI向LUI（语言用户界面）演进。与此同时，其激进的权限管理模式引起了银行及第三方支付应用对其安全性的关注，因而对其施加了不同程度的限制措施。随后，豆包主动下线了其操作手机银行、移动支付等金融类App的能力。“豆包手机”事件推动了跨金融行业对智能体的广泛讨论，使得各方重新审视智能体的权责边界与自主性。

2025年被视为“智能体元年”，智能体逐步从技术概念向产业规模化落地跃迁，成为金融数智化转型的重要驱动力量。2026年以来，技术热点不断涌现。2026年1月，腾讯旗下AI助手“元宝”启动“元宝派”内测。“元宝派”定位为用户的“社交助手”，可参与群聊互动、活跃氛围、执行任务，同时提供影视、音乐等内容推荐，构建“AI融入人际社交”的全新场景。2026年1月底，OpenClaw经过2次改名（Clawdbot、Moltbot），成长为全球知名的开源自主智能体。OpenClaw可基于即时通讯软件接收指令，像“数字管家”一样直接操作电脑文件系统、执行脚本、管理邮件和日程，实现跨平台的自动化任务处理。

智能体技术飞速演进，为行业发展带来重要机遇。在银行业竞争加剧、监管趋严的当下，如何把握技术机遇，安全稳妥有序地拥抱新一代人工智能技术，在合规审慎的前提下发挥智能体技术优势，提高其效率、性能与自主决策能力，是商业银行不容忽视的重要课题。

“自主性”是智能体的关键技术优势

不同于RPA等自动化技术具有标准的工作流程，智能体的优势在于其“自主性”——能够在更为复杂、不确定的环境中进行感知、规划、执行。随着大模型能力不断提升，银行智能体也从早期的“对话智能体”“智能助手”向自主、半自主的智能体架构演进，成为客户的“智能管家”与员工的“办公伙伴”。

自主性直接决定了智能体能否从“问答机器人”或“执行工具”升级为能干活的“数字员工”——不仅“能说会想”，更能高效“执行任务”。Gartner预测，到2029年，客户与银行间25%的客服交互将由“机器客户”完成，智能体将自主解决80%的常见客服问题，无需人工干预，这将使运营成本降低30%。机器将从“顾问”阶段逐步发展为客户的“代理”与“管家”，执行与银行的互动，并具有相应的数字身份。

银行场景具有高度复杂性，同时对结果的准确性、业务连续性有着极为严苛的要求。以手机银行App为例，单个App集成支付、理财、信贷等金融服务，通过支付协议，嵌入生产生活场景。通过线上线下协同和开放生态，在保障安全合规与强风控的前提下，实现获客、活客和留客。相较于上一代人工智能技术，智能体的自主性在银行场景中优势明显。在零售业务中，自主智能体可理解客户诸如“帮我规划下资金”的模糊需求，完成交互渐进式的意图理解，并基于深度推理，自主完成查询、比较、方案生成等一系列操作。在信贷、运营等业务中，智能体可自主串联起复杂流程，触达多个系统和环节，实现从发起、审批到归档的全流程推进。在合规、反诈等领域，智能体可持续监控风险，灵活主动地识别异常情境，并触发相应策略。而在金融机构内部办公场景中，智能体可更为灵活地接管重复性工作，辅助决策，打通更多元的数据链路，实现显著的效率提升。

GUI Agent也是这种“自主性”的典型例子，体现了用户对便捷、无障碍服务的核心诉求。随着智能体的发展演化，这一技术有望成为金融服务“最后一步”的执行者，不断降低交互复杂度，高效、自主完成业务流程。

安全性是金融智能体的必然要求

金融App对豆包手机智能体“超级权限”的限制，是智能体高度“自主”属性与金融行业强安全要求冲突的一个缩影。这一事件表明，资金与隐私安全仍是用户对智能体应用的核心顾虑。

金融智能体应用面临一系列安全挑战，如新型技术风险、模型风险、数据安全风险、合规与科技伦理风险等。在新型网络安全风险方面，智能体交互过程中易遭受提示词注入、对抗样本攻击等威胁，可能引发系统异常或数据泄露。模型方面，可靠性与可解释性仍需加固：模型“幻觉”、输出不稳定及算法偏差可能导致决策失误；在特定业务中，算法“黑箱”难以满足监管要求。数据方面，智能体交易链路的延长进一步放大了隐私数据安全风险敞口。在监管与责任归属方面，智能体的“自主决策”可能导致责任主体模糊。科技伦理方面，数据或算法的偏见可能引发不公平的结果。

金融行业的强监管属性和银行自身稳健的风控策略使得对智能体应用的安全要求极为严苛。这进一步促进了行业、产业界乃至学术界对智能体权限的分级管控，以及数字身份的讨论。金融机构运用该技术的过程中，需在系统设计中严守安全合规架构原则，通过权限控制、多重验证、透明度与可审计设计等策略，推动技术进步，构建全生命周期的安全机制与多层次防御体系，明确智能体的所有权与管理责任，确保安全稳妥、负责任地创新。

银行业智能体架构演进思考

1. 交互入口整合

GUI Agent提供了智能体跨应用执行任务的阶段性方案。在这类技术方案中，智能体可以“看懂”屏幕，进而通过大模型解析用户的自然语言指令，并将其拆解为具体操作步骤——模拟人类的点击、滑动、输入等指令，在屏幕上完成操作，甚至无需API接口。尽管现阶段金融App无法开放大部分GUI权限，使得操作系统级别的GUI Agent还需进一步论证完善，但智能体“超级入口”的发展趋势已然明确。

在消费场景中，产业界已有值得关注的尝试。如千问接入淘宝闪购与支付宝“AI付”，只需下达一句话指令，系统内部的智能体即可迅速调用淘宝闪购下单，无需任何跳转即可完成AI支付。通过“入口整合”，千问正在成为阿里生态的“超级入口”与“总控台”。它打通了淘宝、支付宝、飞猪等核心业务，推动“一句话办事”成为常态。

这一趋势下，传统App导航桌面式的流量分发逻辑将面临根本性挑战，取而代之的可能是更懂用户、更能解决实际问题的“超级入口”或“超级智能体”，并形成分层、分级的智能体调度、协作与管理体系。当然，这一“入口整合”的过程会是一个渐进式的过程。高频、常用、相对不复杂的场景可被快速整合，纳入“超级智能体”路由入口。相对复杂、专业性更强的业务则需进行更为充分的设计优化与安全加固。而入口整合与路由管控也使智能体可监控、可管理，进一步确保其安全性。

2. 复杂智能体协作

新一代智能体的重要特征在于其在复杂情境下的感知、规划与决策能力。随着智能体能处理的业务更为复杂，长流程、复杂协同等问题往往需要多智能体协作（如图1）。这一协作模式可以是顺序、主从、混合协同的。目前，很多金融机构已开展智能体平台的建设，支持智能体的混合调度。智能体协作也是“超级入口”模式的重要支撑，如可以通过“超级客服智能体”路由至产品推荐、远程座席、交易机器人等专业智能体。

图 1 复杂多智能体协作

未来，随着MCP、A2A、ANP等智能体通信协议逐步完善，智能体的协作模式将更为通用、多元，演化为异构、跨平台的多技术栈形态，甚至可能在远期架构中形成智能体生态与网络。需要注意的是，异构智能体的有效整合与高效协同依赖于新兴协议与工具，而不同智能体所依赖的模型也可能是多样且不断演化的。这造成了一定的架构复杂性，也是未来技术演进过程中需优先攻克的难点之一。金融机构可借鉴产业界的相关经验，规划智能体门户与智能体广场，进一步提升智能协同与技术创新。

以GUI Agent、App“超级入口”为代表的下一代应用也将伴随着复杂的智能体交互与工具调用，将系统的安全边界从“接口”扩展到“行为”。银行需为“API+GUI”双通道交互架构做好准备，在开放并管控API的同时，考虑对界面操作的监控管理，并构建必要的追溯机制。

3. 垂域性能提升

如果说“一句话点外卖”这类消费场景的应用依赖智能体广泛的通识能力，企业级垂域智能体应用往往需要更为深入、专精的行业特色知识与能力。随着技术不断进步，垂域智能体的“自主性”也在不断提升，能够在极少人工干预的情况下执行任务，给出初步的业务决策。鉴于金融专业场景的复杂性与强监管要求，多数垂域场景中的智能体仍处于辅助执行的阶段。未来，随着“超级智能体”的发展演化，有望在专业场景中具备真正的“代理”能力，推动银行从被动响应转向主动服务，实现超个性化、千人千面的产品组合与全链路响应支持。例如，专业级的“代理智能体”能够分析客户资产情况，跟踪市场机会，并提供建议，甚至能够作为“客户管家”，与银行进行初步交易协商。这时，客户与人类员工将由“执行者”转变为“监督者”。

鉴于金融专业场景的复杂性以及其在经济体系、民生福祉中的重要性，跨应用、高权限的“超级智能体”可能是由内而外、内外并举的应用与发展路线（如图2）。如在短期内，可主要通过“内部办公助手GUI Agent”完成跨应用的资料整理、文档生成等工作。通过应用系统内的“超级入口智能体”服务客户，不断积累完善垂域能力。目前，以“问数智能体”为代表的数据分析应用已可根据“一句话指令”开展数据检索、预处理，并支持可视化分析，不再深度耦合于企业BI平台。有理由认为，“问数智能体”可进一步延展至ERP、CRM等系统，支撑跨应用的集成。

图 2 由内而外、内外并举的应用路线

智能体垂域性能的提升离不开专业数据资产的积累与垂域模型的迭代。而智能体“动态记忆”与“自我反思”的机制也在不断优化其性能，赋予它们持续学习、性能优化等能力。这需要在智能体架构体系中更多地考虑“记忆”，通过上下文分析、向量数据库存储等组件，完善短期记忆与长期记忆结合的机制，不断积累垂域知识，增强模型性能。记忆机制也便于智能体不断适应用户偏好，提供超个性化的服务。此外，鉴于端侧硬件资源的特点，轻量级的专业化模型也是GUI Agent未来发展所需关注的技术方向。

如需定制化地打造高性能垂域智能体，金融机构也需逐步完善技术栈，积累工程化能力，便于新方法、新工具的引入集成，如优先在内部办公场景落地GUI Agent，可部署专业型的端侧模型，串联高频业务与应用软件。在安全策略、权限管控方面，可基于沙箱部署端侧GUI Agent，为“超级权限”提供安全可信的执行环境。

结语

GUI Agent提供了一类人机协同的范式，体现着智能体逐步从“功能工具”向“自主伙伴”演化。尽管出于强安全考虑，操作系统级别的“超级权限智能体”短期无法在金融场景落地，但这一技术的发展给银行业智能体架构提供了诸多启示。一是交互入口整合，实现端到端、全链路的“对话及服务”落地，从“以产品为中心”转向“以用户为中心”，从“被动响应需求”转向“主动创造价值”。二是异构智能体协同，构筑专业化的“数字员工”协作与管理体系，推动效率升级、能力复用、体验优化、创新加速。三是垂域性能提升，通过数据资产沉淀、垂域模型调优等策略，开展全栈技术体系建设，不断释放专业级的“代理”能力。

从组织模式上看，“人+多智能体协同”可能成为金融智能体应用的重要形式。从技术实现上看，以“xxClaw”“xxbot”为代表的“边云结合+对话式交互(LUI)”可能成为主要的技术形态。我们落地智能体应用时，需随时准备融入智能体生态与网络，实现高价值创造。而人仍然是最终的调度者与监督者，将合规要求与伦理准则嵌入与智能体的交互与决策流程，在创新发展的同时维系“金融为民”的初心使命。
