# 马斯克人在北京，xAI发布首款Coding Agent

> 原文链接：https://mp.weixin.qq.com/s/6M0LqTnR9ufRXB3Zgw8hEw
> 公众号：量子位

鱼羊 发自 凹非寺

量子位 | 公众号 QbitAI

马斯克人在中国，大半夜的还没忘在x上给自家xAI站台。

没错，大洗牌之后，xAI紧锣密鼓地发上重磅新品了——

首个Coding Agent，

Grok Build

。

直接在终端运行、专为专业软件工程和复杂编程任务设计……对标的是谁，属于是摆在明面上的。

这也符合外界对马斯克此前600亿美元收购Cursor的预期——就是要在

Coding

这一块加大力度，追赶Anthropic和OpenAI。

目前，这个Grok Build还是早期测试版，面向SuperGrok Heavy订阅用户开放。

Grok版Claude Code

还是先来看看这个Grok版Claude Code，都主打哪些功能。

第一个重点，是

并行工作的Subagents

（子智能体）。

对于大型、复杂的任务，Grok Build会将工作分派给并行运行的Subagents。

并且这些Subagents的工作是完全透明的。Grok Build在主智能体追踪之上，提供完整的子智能体追踪，并且配备相同的导航控制。

第二个重点，是

兼容现有工作流

。

就是说，你用习惯了Claude Code也没关系，马斯克保证你能无！痛！迁！移！到Grok Build上。

什么AGENTS.md啊，skills啊，MCP servers啊……都能一键导入。在代码仓库中启动Grok Build，它就能立即自动识别并遵循相关开发规范。

另外，Grok Build还有一个

计划模式

。

这是一个针对复杂任务的模式。在正式开始执行代码之前，Grok Build会先征询你的意见，你这个人类项目leader可以针对特定代码留下行内注释，也可以重写整个计划。

计划获得批准执行后，每项更改Grok Build都会以清晰的diff形式显示。

还支持直接在终端中生成图像和视频。

其他的小特性还包括，更快的worktrees；既支持鼠标操作，也兼容vim的操作方式；可以通过“Hunk Tracker”回溯到对话中的任何时间点……

产品调性，正如xAI工程师所说：

我们习惯用vim，习惯用快捷键，并将alt-screen视为主界面。我们的目标是打造一款能够完美融入该工作流的产品，为您提供Grok和终端体验的最佳结合。

不过想要体验确实也不便宜，SuperGrok Heavy的订阅费用是

300刀/月

。

现在打折的情况下，是

前6个月99刀/月

。

动荡的xAI

说实在话，xAI最近在马斯克手上相当不太平。

一方面是联创跑光后持续不断的离职潮——原核心团队一整个分崩离析。

另一方面是马斯克自己也喊话重组，xAI直接并入SpaceX，不再作为独立公司运营。

这不连logo都要焕新了。

但“Grok将死”的声音之中，Grok本身的更新并没有停止。在此次的Grok Build之后，马斯克还透露过，新的Grok模型正在Colossus 2集群上正常训练中。

而随着Cursor团队的加入，xAI换血之后，Grok或许也会呈现新的押注方向。

就看马斯克接下来能搞出什么大新闻了。
