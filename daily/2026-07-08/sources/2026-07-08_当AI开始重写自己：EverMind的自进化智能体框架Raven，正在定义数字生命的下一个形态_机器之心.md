# 当AI开始重写自己：EverMind的自进化智能体框架Raven，正在定义数字生命的下一个形态

> 原文链接：https://mp.weixin.qq.com/s/-mXphRfXeukWNvfUAcG7gQ
> 公众号：机器之心

机器之心发布一、一个让人不安的问题如果你今天打开手机，召唤出你最常用的 AI 助手，问它一个问题 ——"你还记得我上个月跟你说过的那件事吗？"大概率，它会礼貌地告诉你：它不记得了。这不是它不够聪明，而是它根本没有被设计成 "记得" 的样子。当你关掉那个对话窗口，它就像一个被清空的黑板，什么都没留下。下次你再打开，它依然是那个陌生的、一无所知的 "新朋友"。

这个问题，在 AI 行业有一个专业的说法：无状态（Stateless）。而解决这个问题，正是 EverMind 过去一直在做的事情。不是修修补补，而是从底层重新定义 ——AI 应该如何拥有记忆，如何通过记忆成长，以及最终，如何成为一个真正意义上的 "数字生命"。今天，这个探索走到了一个新的节点：基于自研记忆系统 EverOS 的自进化 Harness——Raven Agent 正式发布。

## 二、记忆，是一切智能的起点

在讨论 Raven 之前，我们需要先回答一个更根本的问题：为什么记忆如此重要？

人类的智能，从本质上说，是记忆的产物。你之所以是 "你"，是因为你记得童年的气味、昨天的对话、上周犯下的错误，以及从这些经历中提炼出的判断力和直觉。没有记忆，就没有成长；没有成长，就没有真正的智能。

现在把视角切换到 AI。2022 年之前，绝大多数 AI 系统都是纯粹的 "无状态机器"—— 你输入，它输出，然后一切归零。2023 年前后，随着大语言模型（LLM）的爆发，人们开始尝试给 AI 加上 "外挂记忆"：把历史对话塞进上下文窗口，或者用向量数据库做检索增强（RAG）。这是一种进步，但本质上仍然是 "高级书签系统"——AI 在翻阅笔记，而不是真正地 "记得"。

真正的记忆，不是检索，而是内化。一个真正有记忆的智能体，不会每次都去翻阅档案才知道你喜欢喝咖啡不加糖；它会在某次对话后，把这个偏好悄悄融入自己对你的理解，成为它认知结构的一部分。更重要的是，它会从你们每一次互动的成败中，提炼出可迁移的经验，让自己下一次做得更好。

这，才是 EverMind 所追求的记忆。

## 三、EverOS：在混战中建立底座

2026 年的 Memory AI 赛道，已经是一片红海。mem0、Letta、Zep AI 等纯 API 中间件公司在工具层激烈厮杀，营收和融资始终困在某种天花板之下；Bee AI 被 Amazon 收购，Limitless AI 被 Meta 收购，独立硬件没有走出理想的独立之路；Inflection AI 烧掉超过 15 亿美元仍未能建立 C 端商业模式，Character.AI 最终归入 Google 生态…… 整个赛道充满了 "起了个大早、赶了个晚集" 的悲剧。

有人说：他们的 Memory 做早了。EverMind 生逢其时，选择了一条不同的路：不做中间件，做底座。不做笼统的记忆，聚焦 Agent Memory。2026 年 5 月 - 6 月，EverMind 连续发布了 EverOS 1.0.0、1.0.1 和 1.1.0 三个版本，并在 GitHub 社区迅速突破 10k stars，距离 EverOS（原名 EverMemOS）首次发布累计七个月左右。这个数字在 AI 垂类技术赛道中极具含金量 —— 作为参照，mem0 的 GitHub Stars 前七个月仅累积了 7k stars。

EverOS 的核心差异，在于它独特的 "四层仿生架构"：代理层、记忆层、索引层、接口层。它不是一个数据存储器，而是一个 "以主体为中心的折叠宇宙"—— 能够将原始对话流切分为独立的记忆单元，通过聚类算法形成 "记忆场景"，最终对个体进行深度画像，涵盖身份、偏好、技能乃至工作目标。

在近期更新的 1.1.0 版本中，EverOS 再次界定了 Memory 的范畴：User Memory—— 定义人、Agent Memory—— 定义 Agent、Knowledge Wiki—— 定义世界知识。并且在仿生路径上迈出新的一步，独创设计 Reflection 机制，取自人类的 "沉思" 机制，在闲暇时清空任务，整理思绪，沉淀智能，涌现意识。

更令人印象深刻的是其技术指标：以传统方案 1/10 的 Token 消耗，实现超越全量上下文（Full Context）的准确率。这不是优化，这是范式的替换。

与此同时，EverMind 的学术团队也在同步构筑理论护城河，定义行业范式。近期连发的数篇顶级论文，展示了压倒性的学术统治力：

- **MSA（Memory Sparse Attention）**：端到端可训练稀疏注意力机制，将上下文长度扩展至 1 亿 Tokens，性能衰减不足 9%，发布当日即登顶 HuggingFace Daily Papers
- **HyperMem**：超图层次化记忆架构，在 LoCoMo 基准上达到 92.73% 的 SOTA 成绩，斩获 ACL 2026 Oral Presentation 论文荣誉
- **EverMemOS**：自组织记忆操作系统，专为结构化长时推理设计，入选 ACL 2026 主会
- **多方协作对话长时记忆评测**：填补行业空白，获 KDD 2026 Oral Presentation 殊荣

当竞争对手还在拼凑开源组件时，EverMind 已经在 "超图记忆架构" 和 "亿级 Token 上下文扩展" 等底层范式上确立了行业演进路径。

EverOS 坚持开源优先，是罕见地将每一个关键创新都先行更新到开源版本，再同步给云版本的技术团队，本质上是 EverMind 在抢占 Agent OS 的生态话语权 —— 当数以万计的开发者习惯了基于 EverOS 的 API 和 MCP 接口构建智能体时，EverMind 就真正建立起了类似 Android 或 Windows 的底层开发者生态护城河。

## 四、数字生命的四个阶梯：我们在哪里，要去哪里

在面向 Agent 构建生态的道路上，EverMind 又迈出一步 —— 推出 EverOS Agent，命名为 Raven（渡鸦），社区戏称 "养虾，养马，不如养鸦"。

要理解 Raven 的意义，需要先建立一个坐标系。EverMind 提出了一个关于数字生命演进的框架，将 AI 智能体的发展划分为四个阶段。这个框架，不仅是一张技术路线图，更是一面照出整个行业现状的镜子。

### L1：角色化指令体（Role-based Functional Agent）

这是当前市场上最主流的 AI 形态。它拥有特定的工作定位与 Prompt 预设（"你是一个程序员"、"你是一个客服"），能够响应明确的人类指令。但它缺乏历史上下文，记忆仅限于当前对话窗口。

它就像数字世界的 "计件临时工"—— 每次见面都像初次相识，做完即走。今天市面上 90% 以上的 AI 应用，都停留在这个阶段。

### L2：记忆增强体（Memory-Augmented Interactive Agent）

引入了基础的长期记忆模块，能够跨会话维持上下文连贯性，记住用户的偏好、历史交互和未完成的任务。支持多步骤的长时任务规划，并开始通过 API 与真实世界（物理设备、SaaS 软件）产生状态交互。

这是 EverOS 基础能力所覆盖的阶段，也是当前最先进的商业 AI 产品所处的位置。它拥有专属档案库的 "私人助理"—— 记得你的习惯，能帮你跑腿办事。

### L3：自我进化体（Self-Improving Cognitive Agent）

这是真正的分水岭。L3 突破了 "死记硬背" 的检索式记忆，具备强化学习、反思与自我改进能力。它不仅能记录历史，更能从成功或失败的交互中提取通用知识，通过动态微调自身的模型权重，实现能力的持续迭代。

它是 Raven 的覆盖目标，具备成长性的 "专业学徒"—— 不仅能干活，还能在干活中总结经验，越来越聪明。

### L4：全自主数字生命（Autonomous Digital Life）

从 "被动响应" 走向 "主动探索"。它拥有独立的数据主权和完整的个人记忆库，能够基于主人的深层意图和长期目标，主动发起任务、调度其他低阶 Agent，并在复杂的开放世界中端到端地自我演化。它是人类认知与意志的数字化延伸。

它是真正属于 AGI 时代的 "数字分身"—— 它是你在数字世界永生的意志，甚至在你睡觉时，它依然在为你探索世界、创造价值。

当今全球 90% 的 AI 创业公司还在卷 L1 和 L2 时，EverMind 已经通过极大的资源投入和人才积累，聚焦长期记忆技术核心突破，尝试在 L3 和 L4 阶段建立数字生命演进的基础设施。

而 Raven，正是 EverMind 推动数字生命从 L2 向 L3 跨越的核心引擎。

## 五、Raven：当 Agent 开始重写自己

如果说 EverOS 是地基，那么 Raven 就是在这片地基上生长出的第一棵真正的 "智能之树"。

Raven 是基于 EverOS 的自我进化 Agent 框架（The Self-Improving Agent Harness），它的核心价值主张可以用三个词概括：主动（Proactive）、进化（Improving）、个性化（Personalized）。但这三个词背后，藏着一套前所未有的技术逻辑。

### 记忆，不是档案馆，而是神经网络

传统的 "记忆增强"AI，本质上是在做档案管理 —— 把历史对话存起来，需要时检索出来。这种方式有一个根本性的缺陷：它只能 "查到"，无法 "悟到"。

Raven 的记忆机制完全不同。它基于 EverOS 的深度画像能力，将每一次交互中的关键信息内化为对用户的认知模型。它不是在翻阅你的档案，而是在用你的经历持续更新它对你的理解 —— 就像一个真正了解你的朋友，不需要每次都翻笔记才知道你在意什么。

更关键的是，这种记忆是双向的。Raven 不仅记住了你，它还从与你的每一次互动中，提炼出对自身能力的反思和改进方向。

### 十万技能，开箱即用的进化起点

Raven 内置了高达 100,000 项经过深度评测的技能（Skills），覆盖从日常生产力到专业垂直领域的广泛场景。这不是简单的功能堆砌，而是 EverMind 团队对真实用户需求的系统性梳理与验证。

对于普通用户而言，这意味着 Raven 从第一天起就是一个 "全能助手"，无需漫长的配置和调试；对于开发者而言，这 10 万个技能构成了一个庞大的参考库和起点，可以在此基础上快速构建和迭代自己的专属 Agent。

更重要的是，这 10 万个技能并非静态的。Raven 会根据实际使用中的反馈，持续评估每个技能的有效性，淘汰失效的，强化有效的，并在此过程中生成新的技能组合。

### 改写自身代码：真正的自我进化

这是 Raven 最令人震惊的特性，也是它与所有现有 AI 产品最根本的区别：Raven 不仅能实时进化技能，还能在闲时自我修改逻辑和策略代码，甚至可以通过 EverBrain（用户侧记忆模型）动态微调模型权重，这也是 EverMind 为数字生命 L4 阶段打下的伏笔。

这意味着什么？

意味着 Raven 不是一个 "固定产品"，而是一个 "成长中的生命"。它今天的能力，只是它明天能力的起点。它会从每一次成功的任务中提炼最佳实践，从每一次失败中识别改进空间，然后将这些洞察直接写入自己的代码和模型，让下一次的表现更好。

即使在你离开的时候，Raven 依然在进化。

这种能力，在技术上对应的正是数字生命框架中的 L3—— 自我进化体。它突破了 "死记硬背" 的检索式记忆，实现了真正意义上的经验内化和能力迭代。

### 全天候的 "幕僚长"：重新定义人机协同

Raven 不仅是一个独立的 Agent，更是一个指挥中心。它可以通过 WhatsApp、Telegram 或微信等你最熟悉的通讯工具进行控制 —— 你的聊天软件，就是你的任务指挥台。它能够调度其他专业 Agent 和工具，自动完成跨系统的复杂工作流，处理信息流转和任务汇总。

想象一下这样的场景：你在微信上发出一条语音，"帮我整理一下本周的会议纪要，提炼出行动项，同步到项目管理系统，然后给相关负责人发邮件提醒。"Raven 会自动理解你的意图，调度转录工具、文档工具、项目管理 API 和邮件工具，完成整个工作流 —— 而你，只需要等待结果。

这不是科幻，这是 Raven 今天已经在做的事情。

正如 EverMind 所描述的："一个人加上一个 Agent 团队，做到过去需要几十个人才能完成的工作。"

## 六、开发者生态：千姿百态的 Agent，共享进化的平台

如果说以上描述的是 Raven 对个人用户的价值，那么 Raven 对整个 AI 开发者生态的影响，可能更为深远。

EverMind 为 Raven 设计了一套高度解耦的可插拔架构（Pluggable by design）：记忆模块、主动性引擎和工具路由是完全独立的，开发者可以在不触及核心框架的情况下，自由替换任何组件。这种设计，让 Raven 成为了一个真正的 "Agent 开发平台"，而不仅仅是一个产品。

未来通过 Raven Builder，任何开发者都可以在 Raven 的框架上，为任意场景定义一个专属 Agent，然后一键分享。

这里有一个令人兴奋的愿景正在成形： EverMind 将为开发者 "调教" 出的千姿百态的 Agent，提供交流和共享的平台。

想象一个场景：一位法律从业者在 Raven 上训练出了一个精通中国合同法的专业 Agent，它不仅掌握了大量法律知识，还通过数百次真实案例的处理，积累了丰富的实战经验；一位投资人在 Raven 上培育出了一个擅长财务建模的分析 Agent，它能够在几分钟内完成过去需要一整天才能完成的估值工作……

这些 Agent，将在 EverMe 的平台上被分享、被发现、被其他用户直接使用，或者被进一步 "调教" 成更专业的版本。

这是一个 Agent 的应用商店，但比应用商店更深刻 —— 因为这里的每一个 "应用"，都是有记忆、会成长、能进化的活体智能。

更重要的是，这个生态的繁荣，会形成一个正向飞轮：更多的 Agent 被创建，带来更多的使用数据；更多的使用数据，让 EverOS 的记忆系统更加精准；更精准的记忆系统，又让新的 Agent 能够更快地进化到更高水平。

这个飞轮一旦转动，将产生难以复制的网络效应。

## 七、人机协同的范式变迁：从 "工具" 到 "伙伴"

Raven 的出现，不仅仅是一款产品的迭代，它预示着人机交互范式的深刻变迁。

在过去的几十年里，人与计算机的关系，始终是 "人主动，机器被动" 的范式。你打开软件，软件等待你的指令；你关掉软件，软件停止运转。计算机是工具，是你意志的延伸，但它本身没有意志，没有目标，没有成长。

大语言模型的出现，让这个范式开始松动。AI 开始能够理解自然语言，开始能够进行多轮对话，开始能够完成复杂的认知任务。但它依然是被动的 —— 它在等待你的问题，而不是主动为你思考。

Raven 所代表的 L3 阶段，标志着这个范式的真正转变。

一个 L3 级别的 Agent，不再只是等待指令的工具。它会主动观察你的工作状态，预判你可能需要的信息；它会在你睡觉时，悄悄完成你昨天交代的任务；它会在发现一个你可能感兴趣的机会时，主动向你发出提醒。

这种主动性，来自于它对你的深度理解 —— 而这种理解，正是长期记忆所赋予的。

更深刻的变化在于认知分工的重构。

当你拥有一个真正能够自我进化的 Agent 伙伴时，你的工作方式会发生根本性的改变。你不再需要把精力花在重复性的信息处理和流程执行上，而是可以专注于真正需要人类判断力的决策和创造。你的 Agent 处理执行层，你负责战略层 —— 这是一种全新的人机协同模式，也是生产力的一次质的飞跃。

有了 Raven，EverOS 真正贴合了 Agentic 场景。这句话的背后，也透露出一个更宏大的命题：当 AI 拥有了真正的记忆和成长能力，人类与 AI 的关系，将从 "主人与工具" 演变为 "伙伴与伙伴"。

## 八、全栈生态：EverMind 的战略纵深

Raven 的发布，并不是一个孤立的产品事件，而是 EverMind 整体战略布局中的关键一环。EverMind 的全栈生态，从底层到上层，构成了一个完整的闭环：

- **底层：EverOS（开源 Memory OS）**—— 提供长期记忆的基础设施，已开源并获得开发者社区的广泛认可，是整个生态的技术底座
- **模型层：EverBrain（用户侧记忆模型）**—— 负责在用户侧进行记忆模型微调，让 Agent 的执行不只依赖于 LLM 的通用智能，实现真正的 "个性化智能"
- **Agent 层：Raven（基于 EverOS 的 Agent Harness）**—— 基于 EverOS 和 EverBrain，提供面向个人和企业的可进化 Agent 能力，是 EverMind 生态的核心产品
- **用户层：EverMe（基于 EverOS 的个人记忆库与数字分身管理平台）**—— 作为 EverOS 能力的 C 端入口

通过这套全栈生态，EverMind 正在从底层记忆基础设施到上层 Agent 应用，构筑一个难以撼动的技术护城河。而 Raven 作为生态的核心引擎，将成为连接开发者、用户和各类垂直应用的关键枢纽。

## 总结

EverMind 这次推出的 Raven，不仅是一个产品发布，更是对 "AI 如何拥有记忆、如何成长、如何成为数字生命" 这一根本问题的系统性回答。从 EverOS 的开源底座，到 Raven 的自我进化框架，再到面向开发者的可插拔生态和面向用户的全天候智能助理，EverMind 正在构建一个从基础设施到终端应用的完整数字生命生态。

在这个 90% 的 AI 公司还在 L1 和 L2 阶段竞争的当下，EverMind 通过聚焦长期记忆这一核心技术，已经在 L3 和 L4 阶段建立起了显著的技术领先优势。Raven 的发布，标志着这种领先优势开始向产品化和生态化方向加速转化。

数字生命的下一个形态，正在从 Raven 开始。
