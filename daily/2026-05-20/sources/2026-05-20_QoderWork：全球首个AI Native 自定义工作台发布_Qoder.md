# QoderWork：全球首个AI Native 自定义工作台发布

> 原文链接：https://mp.weixin.qq.com/s/x_8miB_WZA55PKOqrgGvug
> 公众号：Qoder

自 2023 年以来，AI/Agent 产品的核心交互几乎都围绕同一个聊天框展开。这种形态足够简单，也足够具有革命性，因为它第一次把复杂软件的入口变成了自然语言。

但聊天框只是 AI 产业的 1.0 形态：它解决了"入口"问题，没有解决"承载"问题：长文档、设计稿、幻灯片无法在对话气泡里被有效承载，专业工作流的纵深无法被通用对话框表达，知识与品牌也无法跨会话累积。AI 应用形态正在迎来第二次革命：从聊天框 1.0 走向工作台 2.0，从"对话伙伴"走向"专业工作环境"。

因而，当我们把 QoderWork 推向更复杂的场景，一个判断越来越清晰：

自然语言入口依旧必要，但需要搭配具备领域深度的工作场景才能真正释放价值。

为此，QoderWork 推出

自定义工作台

：通过解耦 UI 组件、业务逻辑与工作区区块，产品不再被单一界面形态锁死，完成从「通用对话型」向「领域驱动型」Agentic UI 的升级。

今天，三大首发模式正式集结：设计工作台、幻灯片工作台、写作工作台。

更重要的是，自定义工作台同时是一个开放共建的扩展平台：

未来，任何开发者、任何企业都可以基于同一底座搭建属于自己领域的专属工作台。

设计工作台：说一句话，交付专业设计

设计工作台（Design Desk）是自定义工作台的首个细分领域模式。用户直接描述需求即可在无限画布上获得

可运行、可编辑、可交付

的设计产物。

不同于传统设计工具，它是原生 AI 驱动的「设计即代码」工具：设计文件不再是交付后不可触碰的黑盒，而是团队共同维护的代码资产。通过

Questions

（结构化追问对齐意图）、

Design Plan

（先计划再执行）、

Nudge

（参数化微调）三大机制，设计工作台重构了 AI 生产设计的流程，让"设计稿→标注→沟通→还原"的多轮翻译被压缩到画布之内。

在产业意义上，这是设计交付资产形态的一次根本变革：设计产物从"图层与线条"跃迁到"可运行、可编辑、可交接的代码形态"。设计稿不再是终点，而是一份会持续生长的工程资产，设计师与研发自第一步起便在同一份代码上协作。

幻灯片工作台：把"做一份能讲的幻灯片"

拆解为可控的人机协作流程

自 1987 年 PowerPoint 诞生以来，幻灯片在三十多年里始终是"图层堆叠 + 静态版式"的图形产物，即便加入 AI 直接生成，本质仍是"让模型替你拖图层"。

幻灯片工作台（Slides Desk）想做的事更进一步：把幻灯片从一份图形文件，升格为"可编程、可审计、可累积"的结构化内容资产。这是 PPT 形态自诞生以来的第一次结构性重写。其实现路径是把"做一份能讲的幻灯片"拆解为多阶段协作流程，由 Main Agent 加 6 个专职子 Agent 编排，并以 HTML/CSS 作为渲染中间表达。

全流程可介入。

从上下文采集到内页生成、渲染审校与讲稿生成共 11 个阶段，用户在大纲、主题、图像、内页四个层级都能从容介入。任何写入磁盘的页面都必须先经过用户确认，方向对齐永远在执行之前。

HTML 即源文件。

每一页幻灯片都以独立 HTML 文件落盘，Theme JSON 是视觉的唯一真值，所有页面均从该主题派生。幻灯片内容因此可读、可 diff、可手改、可离线分发；配合内置 Alibaba PuHuiTi 3.0 字体，断网也能完整渲染。

模板沉淀与团队复用。

任意一份完成度足够的幻灯片都可以一键抽取 Theme、装饰花纹与参考页，沉淀为带中文短描述的团队模板。下次新建幻灯片时直接套用模板，品牌视觉研究与主题锁定阶段整体跳过；模板可在团队成员之间共享流转，让品牌调性、汇报体例与版式语言形成可累积的组织资产。

此外，幻灯片工作台内置渲染审校与叙事审校双重机制，自动识别元素溢出、错位与跨页一致性问题；幻灯片完成后支持 PDF、HTML 与 PPTX 三种格式离线导出（PPTX 保留可继续编辑的真实组件，非图片栅格化），并自动为每页生成 150–300 字的演讲备注，让讲解者上场即用。

幻灯片工作台尤其适合销售提案演示、产品复盘汇报、版本迭代说明、行业研究分享、技术评审与课堂教学等需要"既做得专业、又讲得清楚"的演示场景。

写作工作台：让 AI 写作真正变成

"可交付的文档"

在传统聊天式 AI 产品里，长文档常以长长的回复散落在对话气泡之间，复制、修改、归档都极其低效。写作工作台（Writing Desk）将 AI 写作从"聊天里的长答案"升级为"和 AI 一起完成一份可编辑、可批注、可导出的文档"。

专属文档工作台。

进入写作模式，AI 起草的内容直接落到右侧专属的文档工作区。聊天区负责需求与反馈，文档区承载最终内容，从一句话生成框架到逐段打磨收尾，全部在同一工作台内完成。

选中即批注。

这是写作工作台最具差异化的能力。用户可以像审稿一样圈出某段文字，写下"改得更正式""压缩到 100 字以内""翻译成英文"等批注，AI 会围绕被选中的文本定向修改。系统还提供润色、重写、扩写、缩短、修复语法、中英互译等快捷动作。圈出哪里不满意，AI 就改哪里。

人机接力式协作。

写作工作台不是一次性生成器。用户在编辑器里直接调整标题、段落与结构后，AI 会以最新文档为准继续处理，人工编辑与 AI 修改自然接力，避免反复"复述上下文"的内耗。

多轮版本与 PDF 导出。

每一次重要修改都留下历史版本，写作过程可回看、不怕改丢；完成后可一键导出 PDF 用于汇报、归档与外发。系统还提供"正式 / 轻松 / 技术 / 创意"等语气预设，贴合不同的使用场景。

写作工作台尤其适合 PRD、技术方案、活动方案、周报与会议纪要等结构化文本场景。

一个底座，多种形态

设计、幻灯片、写作，三类工作台覆盖了现代知识工作中最高频的视觉设计、汇报材料与结构化文档场景。它们拥有独立的视图、工具与产物标准，又共享同一个 Agent 底座与任务系统。

这正是 QoderWork 自定义工作台想要呈现的图景：

自然语言是入口，工作台是产能。

一个聊天框装不下专业工作，但一个持续生长的工作台体系可以。

这个故事，注定不会由 QoderWork 团队独自书写。设计、幻灯片、写作只是首批内建模式，自定义工作台从第一天起就被设计为开放、可共建的扩展平台：任何开发者、任何企业，都可以基于同一份 Agent 底座、任务系统与扩展机制，构建属于自己业务领域的专属工作台。我们提供底座，行业一起生长。

工作台的时代，才刚刚开始。
